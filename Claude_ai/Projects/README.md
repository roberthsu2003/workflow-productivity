# Projects（專案）：您的雲端專屬知識沙盒

**Projects** 是 Claude 提供的一種**持久化、規模化**的雲端工作方式。如果您覺得一般對話像是在路邊攤快餐，那麼 **Project** 就像是您擁有一間專屬的實驗室。

---

## 🧠 Projects 的本質是什麼？

這是在雲端建立一個 **Sandbox（沙盒）** 嗎？**答案是肯定的，但它是「知識層面」的沙盒：**

1.  **雲端知識沙盒 (Knowledge Sandbox)**：
    - 當您建立 Project，您就在雲端劃出了一個獨立空間。
    - 您上傳的 **Knowledge（知識檔）** 會儲存在這個沙盒中。在這個專案內的所有對話，Claude 都會預先「讀過」這些檔案。
2.  **行為約束器 (Custom Instructions)**：
    - 專案內的「自訂指示」就像是沙盒的物理規則。您可以規定在這個空間裡，Claude 必須扮演什麼角色、輸出什麼格式。
3.  **運算沙盒 (Execution Sandbox)**：
    - 在專案中，您可以啟用 **Analysis Tool**。當需要處理數據或跑程式碼時，Claude 會在雲端的安全沙盒中執行 Python，並將結果回傳給您。

---

## 為什麼要用 Projects 而不是一般對話？

| 特性 | 一般對話 (Chats) | 專案工作區 (Projects) |
| :--- | :--- | :--- |
| **記憶長度** | 單次對話結束即重置 | **持久記憶**（所有對話共用知識檔） |
| **背景設定** | 每次都要重新餵資料 | **一次設定，終身沿用** |
| **隔離性** | 容易受到上一則對話干擾 | **完全隔離**（確保推理不受外界影響） |
| **團隊協作** | 無法直接分享對話脈絡 | **可與小組成員共用**（Team/Enterprise 方案） |

---

## 使用時機

| 適合 | 不適合 |
|------|--------|
| **同一主題**多輪對話（半年報、專案計畫、長期撰寫） | 一次性問答、無需累積脈絡 |
| 需反覆引用**同批 PDF／範本／內規** | 檔案極大或常換主題 |
| 希望 Claude **固定角色與格式** | 需要依任務**動態載入**不同流程（適合 **Skills**） |

---

### 💡 進階技巧：專案隔離 (Project Isolation)
**善用專案隔離提升 AI 效能**：建議針對不同任務建立獨立專案，並僅為其啟用必要的 **Connectors**。這能避免過多的工具定義（Tool Definitions）擠壓系統 Token 空間，防止單一對話的背景資訊過於臃腫，進而維持 AI 的推理精確度與反應速度。

## 經典範例：半年度績效報告專案

### 任務背景
承辦人建立名為「**114 年上半年部門績效報告**」的 Project：已上傳**上層訂頁的格式範本 PDF**、**內部 KPI 定義一頁式說明**，並在自訂指示中要求「語氣正式、須對齊範本章節」。之後每一則對話都在此專案開啟，避免每週重複貼範本。
