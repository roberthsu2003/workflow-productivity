# Projects（專案）

**Projects** 是 **claude.ai** 上的**專用工作區**：可設定**自訂指示（Custom instructions）**、上傳**知識檔（Knowledge）**，讓該專案內**所有對話**共用同一份背景與行為規則。建立與管理見 [How can I create and manage projects?](https://support.anthropic.com/en/articles/9519177-how-can-i-create-and-manage-projects)。免費方案可建立專案數量有限制（以說明中心為準）；Team／Enterprise 可與組織成員共用專案。

**RTCCF 五段欄位**請對照 [Chats／RTCCF 欄位對照](../Chats/README.md#rtccf-欄位對照複製範本時請五段都填)。

---

## 使用時機

| 適合 | 不適合 |
|------|--------|
| **同一主題**多輪對話（半年報、專案計畫、長期撰寫） | 一次性問答、無需累積脈絡 |
| 需反覆引用**同批 PDF／範本／內規**（上傳為專案知識） | 檔案極大或常換主題（應分專案或改用單次上傳） |
| 希望 Claude **固定角色與格式**（在自訂指示寫清） | 需要依任務**動態載入**不同流程（較適合 **Skills**） |

---

## 官方精神（摘要）

專案三要素：**自訂指示**、**知識檔**、**專案內對話歷史**共享同一脈絡；與 **Skills** 的差異可簡記為：專案偏向**持續載入的靜態背景**，Skills 偏向**依任務觸發的動態流程**（見 [Skills 說明](https://claude.com/docs/skills/overview) 中的對照表）。

## 經典範例（辦公情境）：半年度績效報告專案

### 任務背景

承辦人建立名為「**114 年上半年部門績效報告**」的 Project：已上傳**上層訂頁的格式範本 PDF**、**內部 KPI 定義一頁式說明**，並在自訂指示中要求「語氣正式、須對齊範本章節」。之後每一則對話都在此專案開啟，避免每週重複貼範本。

### RTCCF Prompt（在該 Project 內開新對話時使用；請複製後修改）

```markdown
## Role

你是一位熟稔政府或大型組織書面報告的承辦人員。

## Task

依本 **Project 知識中已上傳的範本與 KPI 說明**，撰寫「**{部門名稱}** **{上半年／下半年}** 績效報告」的**初稿大綱與第一版內文**。須對應範本章節順序，並在適當位置標示【待補數據】。

## Context

- 報告讀者：上級機關／董事會（請依實際替換）
- **本階段已知事實或數字（請貼上）：**
  - （貼上已核定預算、實支、辦理場次等）

## Constraint

- 語言：繁體中文；用語與 Project 自訂指示一致
- 不捏造未提供之統計；未確定處用【待補數據】
- 引用範本用語時勿與內部 KPI 定義矛盾

## Format

- 輸出為 **Markdown** 章節結構（可再請開 **Artifact** 方便排版）
- 若需表格，用 Markdown 表格呈現 KPI
```
