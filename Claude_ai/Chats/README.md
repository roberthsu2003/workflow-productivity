# Chats（對話）

在 **Claude.ai 對話**中，你可以先**查詢／彙整**資料（貼上查詢結果、上傳檔案，或搭配 **Connectors** 讀取郵件、行事曆等），再請模型依固定結構產出 **Markdown** 或可下載的 **Word／Excel／PowerPoint／PDF**。下列範例採 **RTCCF**（**R**ole、**T**ask、**C**ontext、**C**onstraint、**F**ormat）撰寫，與本倉庫 [AI 提示詞工程指南](../../prompt/AI提示詞工程指南/README.md) 之「一般任務」五要素一致，方便複製後只改情境與內容。

> **實務提醒**：產出 **.docx／.xlsx／.pptx／.pdf** 時，請確認帳號已啟用 **Skills** 與**程式碼執行**（依方案與介面為準），並在 **Format** 中明確要求「產出可下載檔案」。若當下無法產檔，可先請模型輸出 **Markdown 表格或大綱**，再改請產試算表／文件檔。

---

## RTCCF 欄位對照（複製範本時請五段都填）

| 代號 | 英文 | 建議寫什麼 |
|------|------|------------|
| **R** | Role | 你希望 AI 扮演的專業角色 |
| **T** | Task | 要做什麼、產出物是什麼 |
| **C** | Context | 背景、對象、已查到的資料（可整段貼上） |
| **C** | Constraint | 語氣、長度、不可捏造、隱私與法遵 |
| **F** | Format | 檔案類型、版面、是否用 Artifact／Skill 產檔 |

## 範例一：Markdown（.md）

### 任務背景

承辦人參加跨單位協調會，手邊已有**會議逐字紀錄或重點筆記**，需快速整理成**一則對內周知**，貼在內部群組或知識庫，方便同事用任何文字編輯器或 Git 閱讀。

### RTCCF Prompt（請複製後修改）

```markdown
## Role

你是一位擅長將會議內容改寫成「對內周知」的行政承辦人員。

## Task

將我提供的會議紀錄整理成**單一 Markdown 文件**：含標題、開會資訊、決議事項（條列）、待辦與負責人／期限、下次追蹤點。

## Context

- 會議性質：機關／公司內部協調會（請依實際替換）
- 讀者：未與會同仁與相關承辦
- **以下為查詢／會議原始資料（請貼上或摘要替換）：**
  - （在此貼上逐字稿、語音轉文字、或你自己的重點筆記）

## Constraint

- 語言：繁體中文；語氣正式、簡潔
- 不新增與原始紀錄明顯矛盾的事實；不確定處請標註「待確認」
- 勿輸出與工作無關的寒暄

## Format

- 完整使用 Markdown（`#` 標題、`-` 條列、必要時 `##` 小節）
- 請以 **Artifact** 顯示可複製的全文，或置於單一程式碼區塊外可直接複製的純文字 Markdown
```

## 範例二：Word（.docx）

### 任務背景

需向主管簽核**小型採購或加班補休說明**，內部規定須以 **Word 簽呈**格式留存；你已從內規或往例查到**欄位與簽核順序**，要把本次事由寫成可存檔、可列印的正式文件。

### RTCCF Prompt（請複製後修改）

```markdown
## Role

你是一位熟諳機關／企業內部簽呈格式的行政人員。

## Task

依我提供的背景，撰寫一份可列印存檔的 **Word（.docx）簽呈**：含主旨、說明、金額或時數（如有）、結論與請示事項；預留「簽核欄位說明」文字（不必真的畫表格線亦可，但結構需清楚）。

## Context

- 簽呈類型：（例：物料採購／加班補休／差旅報銷，請自填）
- 主管層級與簽核順序：（例：科長→室主任→副座）
- **查詢／已知事實（請貼上）：**
  - （品項、金額、日期、法規或內規依據摘要）

## Constraint

- 語言：繁體中文；用語與機關／公司慣例一致
- 不捏造核准結果或已發生的簽核；金額與日期須與我提供資料一致
- 若缺關鍵數字，請在文中標註「【請承辦補填】」

## Format

- 請使用 **Word／文件產出相關 Skill**，產出**可下載的 .docx 檔**
- 版面：A4、標楷體或公司規定字型（若無規定則標楷體 12pt）、段落分明
- 若目前環境無法產出 .docx，則改輸出完整 Markdown，並註明可複製到 Word
```

## 範例三：Excel（.xlsx）

### 任務背景

季末需填報**預算執行率**或**採購比價**：已從會計系統或廠商報價**查詢到多筆數字**，要彙成**一張試算表**，方便科長在會議上篩選與排序。

### RTCCF Prompt（請複製後修改）

```markdown
## Role

你是一位熟練 Excel 的會計或採購行政人員。

## Task

將下列資料整理成**單一試算表（.xlsx）**：含欄位名稱、至少一列合計或小計（若適用）、必要時凍結首列。表名與欄位須符合內部報表習慣。

## Context

- 報表用途：（例：Q3 預算執行明細／三家廠商比價）
- **查詢結果或原始資料（請貼上或上傳後在此摘要）：**
  - （科目前綴、預算數、實支數、執行率；或廠商名、單價、交期等）

## Constraint

- 數字須與我提供資料一致；不自行發明金額
- 百分比欄位請保留合理小數位；幣別統一
- 若資料不完整，請在試算表首列加「備註」欄註記缺漏

## Format

- 請優先使用 **Excel／試算表相關 Skill**，產出**可下載的 .xlsx 檔**
- 工作表命名清楚（如「預算執行明細」）；首列為欄名
- 若無法產出檔案，改以 Markdown 表格呈現，並註明可貼入 Excel
```

## 範例四：PowerPoint（.pptx）

### 任務背景

須向主管做**10 分鐘口頭報告**，主題為**業務進度與風險**；你已從專案儀表板或公文**查到關鍵數字與期限**，要做成**5～8 頁簡報**，架構固定為「現況—問題—對策—需求」。

### RTCCF Prompt（請複製後修改）

```markdown
## Role

你是一位擅長對長官簡報的專案承辦人。

## Task

製作一場口頭報告用 **PowerPoint（.pptx）**，約 5～8 頁：封面、現況與數據、課題與風險、因應對策、結論與需要支援事項。每頁重點以條列為主，字數精簡。

## Context

- 聽眾：直屬主管與相關局處／部門長官（請依實際替換）
- 報告時長：約 10 分鐘
- **查詢／已知資料（請貼上）：**
  - （KPI、落後項目、法規或合約期限、資源缺口）

## Constraint

- 語言：繁體中文；一頁一主題，避免滿版文字
- 不誇大績效；不確定數據請標「待確認」
- 避免敏感個資；用職稱或部門代稱

## Format

- 請使用 **PowerPoint／簡報相關 Skill**，產出**可下載的 .pptx 檔**
- 版型簡潔（白底或機關範本色）；標題字級大於內文
- 若無法產出檔案，改輸出每頁的 Markdown 大綱與建議圖表類型
```

## 範例五：PDF（.pdf）

### 任務背景

需對民眾或廠商發布**單頁式申請須知**（或活動報名說明），必須**固定版面、不易被竄改**，以利貼於公布欄或 email 附件；內容已從**主管機關網站或內部函釋**查詢整理完成。

### RTCCF Prompt（請複製後修改）

```markdown
## Role

你是一位擅長撰寫對外說明文件的行政承辦人員。

## Task

將下列要點排版成**單一 PDF 說明文件**（可為 1～2 頁）：含標題、適用對象、應備文件、送件方式與期限、洽詢窗口（可留占位符）。

## Context

- 文件類型：（例：補助申請須知／場地借用／教育訓練報名）
- **查詢／法規與流程重點（請貼上）：**
  - （申請資格、截止日期、地址、電話分機等）

## Constraint

- 語言：繁體中文；對外語氣清楚、無歧義
- 法條與日期須與我提供來源一致；不臆測未公告事項
- 聯絡資訊若未提供，請用「【請補正式窗口】」占位

## Format

- 請使用 **PDF 產出相關 Skill**，產出**可下載的 .pdf 檔**
- 版面：A4、段落與標題層級清楚；必要時加編號步驟
- 若無法產出 PDF，則先給完整 Markdown，並說明可貼到 Word 後另存 PDF
```

## 小結

| 產出 | 典型用途 | 無法一鍵產檔時的備援 |
|------|----------|----------------------|
| Markdown | 周知、版本控管、貼知識庫 | 直接複製對話中的 MD |
| .docx | 簽呈、函稿、正式說明 | 複製 Markdown → Word |
| .xlsx | 報表、比價、名冊 | Markdown 表格或 CSV → Excel |
| .pptx | 口頭簡報 | 大綱貼入 PowerPoint 設計工具 |
| .pdf | 對外公告、固定版面 | Word 或列印另存 PDF |
