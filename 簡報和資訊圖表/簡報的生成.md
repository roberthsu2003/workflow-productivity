# 簡報的生成

本文件以兩條主線整理：**① NotebookLM 產出簡報**、**② PPT 模板取得與套入 PowerPoint**（含與 NotebookLM 的搭配）。其餘工具僅列於文末「選讀」。

**建議總流程：** 先**大綱**（可選）→ **來源文件 + NotebookLM 產出** → **對照 PPT 模板**調整版型／匯出 → 校稿與演練。

---

## 一、NotebookLM 產出簡報

### 1.1 適合什麼情境

| 情境 | 說明 |
|------|------|
| 手邊已有 **PDF／Google 文件／長篇計畫** | NotebookLM 以**來源為準**整理章節，較能對齊原文、降低空談 |
| 要**快速得到簡報初稿** | 一鍵或幾步產出簡報型輸出 |
| 教學、校內簡報、內部報告 | 搭配來源出處較好交代依據 |

### 1.2 操作步驟

1. 建立 Notebook，**上傳**與主題相關的 PDF、Google 文件等**來源**。
2. 在 NotebookLM 中選擇產生 **簡報**（或當前版本提供的簡報／簡報風格輸出；介面名稱可能更新）。
3. 檢視章節是否對應你的文件；可再請它「**每頁重點濃縮至 3 行以內**」「**不要延伸未出現在來源的數據**」。
4. **產出物**：多為**簡報圖像或簡報型版面**，依版本可能**不易直接編輯文字**；新版本若支援以自然語言調整內容，可優先使用官方說明中的編輯方式。

### 1.3 NotebookLM 產出簡報用 Prompt（建議收藏）

<details>
<summary>💬 點開：NotebookLM 產出簡報 Prompt</summary>

```markdown
請依下列規範產出本簡報內容，不更改原始輸入內容：

- 科技感與專業風格，適合高階主管簡報
- 字體全程統一（如思源黑體／標楷體／無襯線，不混用其他字型）
- 每張投影片重點不超過 3 項
- 需附上配色建議（主色、輔色、背景色、字色 HEX），例如：
    - 主色：#20B2AA（藍綠）
    - 輔色：#00FFD0（螢光青綠）
    - 背景色：#1A1D23（深灰黑）
    - 字色：#FFFFFF（白）、#CCCCCC（淺灰）
- 插圖建議：「線條圖示」、「抽象科技感背景」，避免過度複雜或照片
- 內容僅產出重點，文字精簡、不改動原意
```

</details>

### 1.4 優點與限制

| 優點 | 限制 |
|------|------|
| 依**上傳來源**整理，較貼近既有文件 | 產出常為**圖像為主**，若需可編輯 `.pptx` 文字，往往要**另套模板或重打**（見第二節） |
| 省下手動從長文摘重點的時間 | 版型受工具限制，**品牌模板**需到 PowerPoint 再對齊 |

**完成檔參考：** [NotebookLM 簡報產出範例](./簡報完成檔案)（依你專案內實際檔案為準）。

> **提示**：NotebookLM 功能更新頻繁，按鈕名稱以 [官方說明](https://support.google.com/notebooklm) 與當前介面為準。

## 二、PPT 模板產出與套用（PowerPoint）

### 2.1 為什麼要「模板」

- **視覺一致**：封面、內頁、標題層級、色票統一，像同一套簡報。
- **省時間**：不用從白投影片開始拉線條與對齊。
- **可編輯**：下載的 `.pptx` 在 **Microsoft PowerPoint**（或相容軟體）裡**逐頁改字、改圖**，符合交付需求。

### 2.2 模板哪裡找（參考來源）

- [Slidesgo](https://slidesgo.com/)
- [Slidebean](https://slidebean.com/)
- [Slides Carnival](https://www.slidescarnival.com/)
- 靈感參考（注意版權）：[Pinterest](https://www.pinterest.com/)、[Behance](https://www.behance.net/)

挑選時先決定：**深／淺色底**、**是否含圖表頁**、**頁數是否夠用**，再下載 PowerPoint 格式。

### 2.3 設計元素與版權提醒

若從網路模板**截圖**當「靈感」：

- 請**不要直接抄襲**有版權的完整版面作為商業重製。
- 可記錄**配色 HEX、區塊比例、字級層級**，在自己的模板或 NotebookLM 提示中**重新描述**。

**本專案範例圖（封面／內頁參考）：** [簡報範例元素 source](./簡報範例元素source/)

### 2.4 把 NotebookLM 與 PPT 模板串在一起（推薦工作流）

這是兩條主線的**合流**：**模板負責版型與品牌感**，**NotebookLM 負責依文件產出內容與視覺初稿**。

**流程：**

1. **選定並下載**一套 `.pptx` 模板（封面 + 章節 + 內容頁版型）。
2. 將**計畫書 PDF**（與需要時的**設計參考圖**）上傳 NotebookLM，產出簡報。
3. 在 **PowerPoint** 中**開啟模板**，將 NotebookLM 各頁的**標題與三點重點**複製到對應版型；若 NotebookLM 給的是整張圖，可**插入為圖**或**只抄文字**到模板文字框（依清晰度與編輯需求決定）。
4. 統一字體與色票與模板一致（例如全用思源黑體）。

<details>
<summary>💬 搭配用 Prompt（上傳 PDF + 可選設計參考圖時）</summary>

```text
使用來源中的圖片做為投影片背景，並使用來源中的文字做為投影片文字。
1. 內文頁的部份要避免重複使用同一背景
2. 字體固定使用思源黑體
3. 文字和圖片配色要與背景一致
```

</details>

若介面中**沒有**「簡報」按鈕，可先請 NotebookLM 產出**條列式綱要（每頁標題 + 三點）**，再**手動貼入** PowerPoint 模板。

## 三、先確定大綱（可選，但建議）

沒有長文件、只有主題時，可先用 AI 產**大綱**，再餵給 NotebookLM 或自己逐頁填入模板。

### 範例：科技感主題簡報大綱

**主題背景**：你是**華梵大學資訊處**承辦人，需向校務會議簡報「**2025 校園數位轉型計畫**」，內容含 AI 工具導入、智慧教室建置、資安升級。希望產出約 **8 頁**簡報大綱，風格偏科技感。

<details>
<summary>💬 建議 Prompt（ROSES 框架）</summary>

```markdown
## R – 角色設定
你是一位擅長科技主題簡報的簡報顧問，熟悉數位轉型、AI 應用等議題。

## O – 任務目標
幫我產出一份「2025 校園數位轉型計畫」的簡報大綱，約 8 頁，風格偏科技感。

## S – 執行步驟
1. 封面頁（標題、副標、報告單位、日期）
2. 6～8 張投影片，每張有標題與 3 個重點
   - 建議主題：現況分析、AI 工具導入、智慧教室、資安升級、時程規劃、資源需求、總結
3. 結尾頁（總結或 Q&A）

## E – 範例格式
第 1 張：封面
第 2 張：校園數位轉型現況
  - 重點 1、重點 2、重點 3
（以此類推）

## S – 範圍與風格
- 語言：繁體中文
- 風格：科技感、專業、適合向高階主管簡報
- 每張投影片重點不超過 3 項
```

</details>

## 四、風格與配色（簡要）

| 項目 | 建議 |
|------|------|
| **配色** | 深色底 + 霓虹藍／青綠／電紫點綴；或白底 + 單一科技主色（如 #0066CC） |
| **字體** | 標題無襯線粗體（如思源黑體 Heavy）；內文易讀、少混用字體 |
| **原則** | 一頁一重點（One Slide, One Idea）；對比清楚 |

**自然語言範例：** 「整份簡報採深色背景、主色藍綠漸層，標題白色，輔助文字淺灰，圖表用螢光青綠強調。」

## 五、選讀：其他產出方式

當你的重點**不是** NotebookLM 與 PPT 模板時，可參考：

| 方式 | 說明 |
|------|------|
| **Gemini Canvas** | 可產出網頁式簡報；若必須 **`.pptx`**，仍需匯出或複製到 PowerPoint 模板。 |
| **ChatGPT Canvas** | 可產出 **`.pptx`** 時，常為**偏純文字**，視覺仍建議套用第二節模板。 |
| **手動（PowerPoint／Keynote／Google Slides）** | 自由度最高，但從零排版與對齊最耗時；**建議仍下載模板**再填內容。 |

**手動工具列舉：** Microsoft PowerPoint、Mac Keynote、Google Slides、Canva、Gamma 等。

## 六、需求快速對照（以本文件主軸為準）

| 你的需求 | 建議 |
|----------|------|
| 有長 PDF／報告，要快出簡報初稿 | **第一節 NotebookLM** |
| 要可交給長官、版型專業、可改字 | **第二節 PPT 模板** + 將 NotebookLM 文字／圖套入 |
| 沒有文件、只有主題 | **第三節 大綱** → 寫成短文件或逐頁手打，再視需要餵 NotebookLM |
| 一定要 `.pptx` 且版面精美 | **模板下載**為主，NotebookLM 為內容來源 |
