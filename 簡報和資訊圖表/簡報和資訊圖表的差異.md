# 簡報 vs. 資訊圖表：核心差異與應用指南

在視覺傳達的領域中，「簡報」（Presentation）與「資訊圖表」（Infographic）常被混淆。雖然兩者都運用視覺元素來傳遞訊息，但其**設計邏輯**與**溝通本質**卻大相徑庭。

## 1. 核心定義與定位

### 簡報 (Presentation)

- **定位：** 講者的輔助工具。
- **本質：** 線性的敘事過程。它通常需要「人」的解說才能完整，簡報本身是配角，講者才是主角。

### 資訊圖表 (Infographic)

- **定位：** 獨立的資訊媒介。
- **本質：** 非線性的探索過程。它必須具備「自明性」（Self-explanatory），即便沒有解說者，讀者也能自行讀懂內容。

## 2. 深度維度對比表

|           |                        |                          |
| --------- | ---------------------- | ------------------------ |
| **維度**    | **簡報 (Presentation)**  | **資訊圖表 (Infographic)**   |
| **資訊密度**  | **低密度**。強調單一焦點，避免觀眾分心。 | **高密度**。將大量數據與邏輯壓縮在單一畫面。 |
| **敘事結構**  | **線性**。由講者掌控翻頁，有前後順序。  | **非線性**。讀者可自由跳讀，自行發現關聯。  |
| **存在形式**  | 多頁幻燈片 (Slides)。        | 單一長圖、海報或網頁版面。            |
| **閱讀主動性** | 聽眾是被動的，節奏由講者控制。        | 讀者是主動的，可自由決定停留時間。        |
| **文字量**   | 極少。僅保留標題與關鍵字。          | 中等。包含必要的數據標籤與短敘述。        |
| **生命週期**  | 演講結束後，其價值隨之大幅下降。       | 具備長效傳播性，適合社群分享與參考。       |

## 3. 視覺設計邏輯

### 簡報：為了「看清」

1. **大字體：** 確保最後一排觀眾也能看見。
2. **視覺焦點：** 每頁只說一件事（One Slide, One Idea）。
3. **情感連結：** 大量的圖像與留白，用來營造氛圍或情緒。

### 資訊圖表：為了「看懂」

1. **視覺層級：** 透過大小、顏色區分資訊的重要程度。
2. **邏輯線索：** 使用箭頭、線條或區塊引導讀者的視線流動。
3. **數據轉換：** 將枯燥的數字轉化為直觀的圖示（如：用 10 個小人圖代表 10%）。

## 4. 互換與轉化：資訊的「原子化」與「模組化」

簡報與資訊圖表並非互斥，它們可以透過「解構」與「重組」進行互相轉換。

### 4.1 資訊圖表拆解為簡報（由大到小：放大鏡原則）

將一張高密度的長圖拆解為 10-15 頁簡報，適合**深度教學**。

- **拆解邏輯：** 找出圖表中的「視覺模組」。每一個數據區塊、每一個步驟或每一個邏輯層級，都可以獨立成為一頁簡報。
- **轉化步驟：**
    1. **擷取局部：** 將圖表中的關鍵圖示或圖表擷取出來。
    2. **加入敘事：** 資訊圖表原本只有文字標籤，轉化為簡報時需加入講者的導言。
    3. **動態展示：** 透過簡報動畫（如淡入、移動），讓圖表中的元件依序出現。

### 4.2 簡報組合成資訊圖表（由小到大：導覽圖原則）

將 30 頁的線性簡報濃縮為一張資訊圖表，適合**課後複習**或**社群傳播**。

- **組合邏輯：** 找出所有頁面的「邏輯關聯」。捨棄多餘的裝飾性圖片與轉場文字，僅保留結論與核心數據。
- **轉化步驟：**
    1. **關鍵提取：** 每一頁簡報只提取一個最核心的數據或金句。
    2. **視覺地圖化：** 決定資訊流向（Top-Down、Z-Pattern 或 S-Curve），將原本分開的頁面用視覺線索連起來。
    3. **層級重整：** 將最重要的結論放大，次要的數據縮小，建立視覺層級。

## 5. 應用場景與建議

### 什麼時候該用「簡報」？

- **商業提案：** 需要透過肢體語言與口氣來打動客戶。
- **內部週報：** 重點在於溝通下週計畫，而非呈現所有細節。
- **主題演講 (Keynote)：** 目的是啟發觀眾思考。

### 什麼時候該用「資訊圖表」？

- **社群媒體傳播：** 需要在幾秒鐘內抓住使用者的注意力。
- **操作指南 (Manual)：** 將複雜的步驟（如 n8n 流程）具象化，供使用者對照。
- **年度報告精華：** 將 50 頁的數據報告濃縮成一張精華圖。

## 6. 常見的錯誤：誤把資訊圖表當簡報

這是最常見的「災難性」設計：

- **現象：** 講者將密密麻麻的資訊圖表直接貼在 PPT 上。
- **結果：** 觀眾會忙著讀圖上的小字，完全聽不進講者的話。這會造成嚴重的「認知過載」。

**解決方案：**

如果你必須在簡報中引用資訊圖表，請**將其拆解**（參考 4.1 節）。將圖表的各個部分分頁展示，或者僅保留最核心的結論圖表，其他的作為課後附件提供。

## 總結

- 如果你想**主導**對話，請用**簡報**。
- 如果你想**傳遞**知識並讓人隨時查閱，請用**資訊圖表**。
