# 資訊圖表的生成

**資訊圖表（Infographic）**是在**單一畫面**裡，用圖像、圖示與精簡文字，把複雜資訊整理成可快速理解的視覺內容；常見用途包括社群長圖、操作指南、年度報告精華等。

> 與「簡報」的差異（一頁一重點 vs. 一圖多邏輯）請見：[簡報與資訊圖表的差異](./簡報和資訊圖表的差異.md)。

---

## 建議工作流程

1. **釐清目的與受眾** — 誰要看？要「記住一句話」還是「照著步驟做」？
<details>
<summary>範例</summary>

- 年度報告摘要：希望全公司同仁能「記住年度三大成果」。
- 操作指引：讓新手可以「跟著步驟完成設定」。
- 社群宣傳：讓粉絲「一眼明白活動亮點」，加深轉發動機。
</details>

2. **準備內容與資料** — 事實、數字、步驟、因果關係；優先使用**可引用來源**。

<details>
<summary>範例</summary>

- 事實數據：2023年公司營收成長 15%；使用者數超過 10 萬人。
- 步驟拆解：1. 登入系統 → 2. 選擇服務 → 3. 完成設定 → 4. 確認結果。
- 因果關係：銷售下滑主要因市場競爭加劇，導致市占率下降。
- 來源列舉：數字與內容請標明出處（如官方統計、年度報告、學術論文等）。

</details>

3. **選工具產出初稿** — 見下節「常用 AI 工具」。

4. **自訂風格與閱讀動線** — NotebookLM 可指定風格與 F/Z/S 動線；其他工具则用 Prompt 或模板。

5. **迭代微調** — 縮短閱讀時間、檢查是否誤導、統一字級與配色。

## NotebookLM：風格、動線與使用要點

NotebookLM 可依你上傳的資料產生**資訊圖表**。較新版本支援在產生前調整**視覺風格**與**閱讀動線**，減少「千篇一律的 AI 感」，並讓版型更符合場景（學術、商務、社群等）。

### 1. 從「隨機」到可自行定調

- 在資訊圖表相關選項旁，透過**編輯／鉛筆圖示**（介面可能隨版本變更）進入風格選單。
- 依**聽眾**切換風格：對主管偏正式、對學生可偏活潑或手繪感。

### 2. 簡報 vs. 資訊圖表（避免用錯邏輯）

| 類型 | 設計重點 |
|------|----------|
| **簡報** | 一頁一個核心訊息，字少、給講者補充 |
| **資訊圖表** | **單一畫面**承載較多結構化資訊，讀者**沒人講解也要看得懂** |

**視覺化陷阱**：若讀者「看圖花的時間」比「讀短文字」還久，就失去意義。視覺化是為了**加速理解**，不是只為好看。

### 3. 三種常見閱讀動線（可在 Prompt 裡指定）

| 動線 | 說明 | 適用情境 |
|------|------|----------|
| **F 型**（上到下／左到右） | 最常見的線性掃描 | 一般說明、條列式內容；未特別指定時多為此類 |
| **Z 型** |  eyes 習慣「之字」掃過畫面 | **對照、現況 vs. 目標、四象限**（例：左上現況、右上挑戰、左下方案、右下成果） |
| **S 型**（蛇形） | 長流程在畫面上轉折往下 | **步驟多**（例如 7～8 步教學）；直式長圖常有較佳空間運用 |

**Z 型 Prompt 範例：**

```text
請採 Z 型動線設計資訊圖表：左上角「現狀」、右上角「挑戰」、左下角「解決方案」、右下角「預期成果」。內容需完全依我上傳的來源文件，不要捏造數據。
```

### 4. 風格選用提醒（示例）

以下為常見風格類型之**策略性用途**（實際名稱以 NotebookLM 介面為準）：

| 風格傾向 | 較適合 |
|----------|--------|
| 手繪感 | 親和力、通識／生活化主題 |
| 專業／商務 | 對外提案、正式報告 |
| 科學／技術感 | 數據、流程、需要「結構感」的議題 |

⚠️ **版面限制**：若介面中有**固定版型**的風格（例如部分產品中的「社論風」「便當格（Bento）」式），可能**不適合再強行疊加自訂 F/Z/S 動線**，否則容易版面衝突；可改選較自由的風格再下 Prompt。

### 5. 組合建議（口訣）

- 需要**強對照** → 可試 **Z 型 + 清晰區塊或手繪標註感**。
- **多步驟教學** → 可試 **S 型 + 較活潑或易讀的風格**。

> **注意**：Google 產品更新頻繁，選單名稱與項目數可能變動；以 [NotebookLM 官方說明](https://support.google.com/notebooklm) 與當前介面為準。

## 資訊圖表的內容與資料

產圖前先準備「可被圖像化」的材質：

1. **核心訊息一句話** — 這張圖讀完應記住什麼？
2. **結構** — 時間軸、比較表、流程步驟、因果、階層（樹狀）擇一或組合。
3. **數據** — 出處、單位、是否需四捨五入；避免 AI **無來源捏造**。
4. **禁區** — 個資、未公開標案細節、有版權爭議的圖表截圖。

**範例（給自已或給 AI 的素材表）：**

| 區塊 | 內容 |
|------|------|
| 標題 | 2025 校園節能三計畫 |
| 副標 | 依總務處公開簡報整理 |
| 區塊 A | 現況：用電較前年 +3% |
| 區塊 B | 措施：LED 汰換、空調排程、教育宣導 |
| 區塊 C | 目標：年底降載 8%（示意，非真實數字） |

## 找尋資訊圖表的風格與配色

1. **跟著品牌** — 學校／機關 CIS：主色、輔色、是否禁用漸層。
2. **跟著媒體** — LINE 長圖、Instagram 方圖、PDF 海報，**長寬比**先決定再設計。
3. **參考不抄襲** — Pinterest、Behance、政府開放資料視覺化案例；取「配色與層級」，不直接複製版式與插畫。
4. **對比與無障礙** — 文字與背景對比足夠；避免紅綠皆憤單靠顏色傳達差異（可加圖示或標籤）。

**一句話 Prompt 範例（給圖像生成工具）：**

```text
扁平化資訊圖表風格，主色深藍 #1a365d、輔色青綠 #319795，白底，高對比大字，無襯線體，適合列印 A4。
```

## 使用 Nano Banana Pro 生成資訊圖表

專案中常將 **Google Nano Banana Pro** 用於**圖表感、插畫感**的視覺素材（實際入口與方案以 Google 當前產品為準）。

**建議流程：**

1. 先用文字或表格整理好**要呈現的欄位與數字**（避免模型自由發揮編故事）。
2. Prompt 中寫明：**風格**（扁平／等距／手繪）、**長寬比**、**主色**、**不要內嵌無法驗證的具體統計**若你沒提供數據。
3. 匯出後在 **Canva／Affinity** 等工具中**排版、加標題與來源小字**。
4. 若與 NotebookLM 並用：NotebookLM 出**結構與文案初稿**，Nano Banana Pro 出**裝飾性圖示或情境圖**，再合成。

**簡短 Prompt 範例：**

```text
16:9 橫幅，資訊圖表插畫，四個步驟由左到右箭頭連接：上傳檔案 → AI 分析 → 產生重點 → 匯出分享。配色藍與灰，扁平化，不要有真实人脸。
```

## 迭代方式微調資訊圖表

| 階段 | 你可以做的事 |
|------|----------------|
| **第一輪** | 只檢查**事實與邏輯**是否錯、有無幻覺數字 |
| **第二輪** | 刪字：標題更短、bullet 改為詞組、拿掉重複圖 |
| **第三輪** | 動線：請 AI「把 3、4 步對調以符合時間順序」或改 F → Z |
| **第四輪** | 一致：字級層級、色系、圖示風格統一 |

**給 NotebookLM 的迭代 Prompt 範例：**

```text
在維持來源事實的前提下，將這張資訊圖表改為：① 總字數減少 30%；② 標題改為問句式以吸引閱讀；③ 維持 Z 型動線。
```
