# 資訊圖表的生成

**資訊圖表（Infographic）**是在**單一畫面**裡，用圖像、圖示與精簡文字，把複雜資訊整理成可快速理解的視覺內容；常見用途包括社群長圖、操作指南、年度報告精華等。

> 與「簡報」的差異（一頁一重點 vs. 一圖多邏輯）請見：[簡報與資訊圖表的差異](./簡報和資訊圖表的差異.md)。

---

## 主軸流程：Nano Banana Pro → Canva / Affinity

本文件以 **Google Nano Banana Pro**（專案中常用稱呼；實際產品名稱與入口以 Google 官網為準）**產出視覺初稿**，再以 **Canva** 或 **Affinity** **修改內容、排版與細節**，完成可交付的資訊圖表。

| 階段 | 工具 | 你在做什麼 |
|------|------|------------|
| ① 定義內容與結構 | 文件／試算表／自己整理的表 | 核心訊息、區塊、數字與出處（避免 AI 幻覺） |
| ② 生成主視覺與圖像元素 | **Nano Banana Pro** | 依 Prompt 產出全圖或分區素材（背景、圖示、情境插畫） |
| ③ 改字、對齊品牌、匯出尺寸 | **Canva** 或 **Affinity** | 替換錯字、重排層級、去背、修掉亂字、加 logo／來源 |

---

## 建議工作流程（細部）

1. **釐清目的與受眾** — 誰要看？要「記住一句話」還是「照著步驟做」？

<details>
<summary>範例</summary>

- 年度報告摘要：希望全公司同仁能「記住年度三大成果」。
- 操作指引：讓新手可以「跟著步驟完成設定」。
- 社群宣傳：讓粉絲「一眼明白活動亮點」，加深轉發動機。

</details>

2. **準備內容與資料** — 事實、數字、步驟、因果關係；優先使用**可引用來源**。

<details>
<summary>範例</summary>

- 事實數據：2023 年公司營收成長 15%；使用者數超過 10 萬人。
- 步驟拆解：1. 登入系統 → 2. 選擇服務 → 3. 完成設定 → 4. 確認結果。
- 因果關係：銷售下滑主要因市場競爭加劇，導致市占率下降。
- 來源列舉：數字與內容請標明出處（如官方統計、年度報告、學術論文等）。

</details>

3. **Nano Banana Pro 產圖** — 先求「構圖、風格、大致區塊對」，文字可在下一步用 Canva／Affinity 修正（生成圖上的字常需改）。

4. **Canva 或 Affinity 修改內容** — 依交付物選工具（見下節「分工對照表」）。

5. **迭代** — 若 Nano Banana Pro 整張重生成成本高，可只重生成「問題區塊」的裁切區，再貼回 Canva／Affinity 合成。

## Canva vs. Affinity：修改內容時怎麼選？

| 面向 | **Canva** | **Affinity**（Designer / Photo） |
|------|-----------|----------------------------------|
| **定位** | 線上模板、快速排版、多人協作 | 本機專業排版／影像修圖，檔案 fully 自控 |
| **改文字** | 拖曳文字框、品牌套件、範本置換快 | 精確字距／字級、印刷轉曲、PDF 印前 |
| **改圖／修瑕疵** | 基礎去背、部分「魔術」替代較輕量 | **修掉錯字／髒邊／接縫**、修復、進階選取較強 |
| **從 Nano 匯入** | 匯入 PNG／JPG → 上層疊加新文字 | 匯入後分層困難時，常用「**上層蓋住重寫字**」或局部重匯入 |
| **適合誰** | 教材、社群尺寸、非設計背景求快 | 要細修像素、印刷、或已熟 Affinity 流程者 |

實務上常見組合：**Nano Banana Pro 出圖 → Canva 快速換字與套品牌 → 若畫面上有改不掉的 AI 錯字，再進 Affinity Photo 修掉或蓋圖**。

## 使用 Nano Banana Pro 生成資訊圖表

### 原則

1. **先結構後畫面**：用一小張「區塊示意」（紙上或表格）寫好 A/B/C 區要什麼，再寫 Prompt；避免模型自由發揮故事。
2. **長寬比先講清**：直式長圖（如 9:16、適合 LINE）、橫幅（16:9）、方圖（1:1）**開頭就寫在 Prompt**。
3. **數字要麼自己給，要麼不要寫死**：未提供數據時，請模型用「placeholder／示意文字」或純圖示，避免假統計。
4. **風格關鍵詞**：扁平／等距／孟菲斯／玻璃擬態／手繪線稿／瑞士網格擇一即可，疊太多風格易糊。

### Prompt 範本（可複製改寫）

```text
【用途】資訊圖表單張，繁體中文情境。
【比例】寬 1920 x 高 1080，橫式。
【結構】左到右四欄：①問題 ②分析 ③作法 ④成果；每欄一個簡單圖示，欄位之間用箭頭連接。
【風格】扁平化、主色 #1a365d 與輔色 #319795、白底大量留白、無襯線字感。
【禁止】不要真實人臉、不要無來源的具體百分比數字；若需數字請用「XX%」佔位。
【標題區】頂部留白放標題「——」（我會在 Canva 裡用正確標題覆蓋）。
```

### 進階：分區生成再拼接

當整張一次生成常出錯字或擠版時，可拆成：

- 圖一：只生成「頂部標題區背景與裝飾」  
- 圖二：只生成「中央流程圖插畫，文字用極少或英文 placeholder」  

再在 **Canva** 或 **Affinity** 裡對齊裁切、疊上正確繁體中文。

### 給 Nano Banana Pro 的迭代指令（範例）

```text
維持上一張的配色與比例，只改中間第三欄：把「作法」改成三個小步驟圖示（下載→登入→完成），其餘兩欄不要動。
```

## 使用 Canva 修改內容（接在 Nano Banana Pro 之後）

**典型步驟：**

1. **建立畫布** — 選擇與 Nano 輸出一致或略大的尺寸；社群可套用官方建議尺寸。
2. **匯入 Nano 圖** — 設為底圖或鎖定底層；**在上層**加文字框覆蓋錯字、補標題與副標。
3. **品牌** — 套用「品牌套件」：字體、色票、logo；不要直接在 raster 底圖上硬拉變形。
4. **可讀性** — 檢查標題階層（H1/H2）、對比度；長圖用**對齊線**與一致左右邊距。
5. **匯出** — 網頁／社群用 PNG 或 JPG；要印刷時確認解析度（dpi）與出血（若需要則改以 Affinity 收尾）。

**Canva 擅長的「修改」：** 快速替換文案、複製多版本尺寸（resize）、團隊留言修正、套用現成資訊圖表範本後「只換你的 Nano 底圖」。

## 使用 Affinity 修改內容（接在 Nano Banana Pro 之後）

**Designer（向量／版面）適合：** 精確對齊、多頁 PDF、轉曲與印刷輸出、在向量環境補簡單圖示與線條。

**Photo（像素修圖）適合：** **塗掉 AI 圖上錯字或亂符號**、修邊、調光、修接縫、局部銳利化。

**典型步驟：**

1. **置入** Nano 匯出的 PNG／JPG（Photo 或 Designer 皆可，依你偏好的流程）。
2. **文字覆蓋** — 若底圖有錯字：用選取工具與修復／仿製蓋掉原字，再以文字工具在**新圖層**打正確繁體中文（比硬擦半透明底更乾淨）。
3. **裁切與對齊** — 依最終長寬裁切；留意社群安全區（不被裁掉重要訊息）。
4. **輸出** — PDF（印刷）、PNG（透明去背元素若你在 Designer 處理向量）。

**Affinity 擅長的「修改」：** 細修像素瑕疵、控制印刷與轉檔、不希望內容留在雲端編輯器時的流程。

## 資訊圖表的內容與資料

產圖前先準備「可被圖像化」的材質：

1. **核心訊息一句話** — 這張圖讀完應記住什麼？
2. **結構** — 時間軸、比較表、流程步驟、因果、階層（樹狀）擇一或組合。
3. **數據** — 出處、單位、是否需四捨五入；避免 AI **無來源捏造**。
4. **禁區** — 個資、未公開標案細節、有版權爭議的圖表截圖。

**範例（給自已或給 Nano Banana Pro 的素材表）：**

| 區塊 | 內容 |
|------|------|
| 標題 | 2025 校園節能三計畫 |
| 副標 | 依總務處公開簡報整理 |
| 區塊 A | 現況：用電較前年 +3% |
| 區塊 B | 措施：LED 汰換、空調排程、教育宣導 |
| 區塊 C | 目標：年底降載 8%（示意，非真實數字） |

## 找尋資訊圖表的風格與配色

1. **跟著品牌** — 學校／機關 CIS：主色、輔色、是否禁用漸層。
2. **跟著媒體** — LINE 長圖、Instagram 方圖、PDF 海報，**長寬比**先決定再設計。
3. **參考不抄襲** — Pinterest、Behance、政府開放資料視覺化案例；取「配色與層級」，不直接複製版式與插畫。
4. **對比與無障礙** — 文字與背景對比足夠；避免**紅綠色盲**讀者僅靠顏色分辨（可加圖示或標籤）。

**一句話（寫進 Nano Banana Pro Prompt）：**

```text
扁平化資訊圖表風格，主色深藍 #1a365d、輔色青綠 #319795，白底，高對比大字，無襯線體，適合列印 A4。
```

## 迭代方式微調

| 階段 | 建議 |
|------|------|
| **事實** | 數字、名詞、是否牴觸你的來源 |
| **可讀性** | 標題是否過長；是否在 Canva 加「遮色片」讓底圖不搶字 |
| **視覺** | Affinity 修掉毛邊／錯字；必要時只重跑 Nano 的某一裁切區 |
| **一致** | 全系列多張圖時，鎖定同一套色票與字級規則（Canva 品牌套件尤佳） |

**在 Canva 裡「小改不重画」：** 複製頁面 → 只改文案與圖示，底圖沿用。  
**在 Nano 裡「大改構圖」：** 調整 Prompt 後整張重生成，或改為分區生成。
