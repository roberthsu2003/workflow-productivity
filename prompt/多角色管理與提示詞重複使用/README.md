# 多角色管理與提示詞重複使用

當你需要**多個不同角色**（例如：客服助手、會議記錄助手、程式碼助手），又希望**重複使用**這些提示詞、不用每次重寫時，主流 AI 工具都提供對應功能。以下分別說明 **Gemini**、**ChatGPT**、**Claude** 的管理方式。

---

## Gemini 的 Gem 功能

### 什麼是 Gem？

**Gem** 是 Gemini 網頁版（[gemini.google.com](https://gemini.google.com)）內建的「可重複使用的角色模板」。每個 Gem 都預先設定好一套系統提示詞，代表一個固定的角色或任務，點選後即可切換使用。

### 常見的 Gem 範例

| Gem 名稱 | 用途 |
|----------|------|
| 資訊圖表規畫師 | 規劃資訊圖表的結構與內容 |
| 創意漫畫家 | 生成創意漫畫相關內容 |
| 程式說明書助理 | 協助撰寫程式說明文件 |
| 程式教學輔助助理 | 輔助程式教學、解釋程式概念 |
| 迭代 prompt 生成器 | 協助迭代優化提示詞 |

### 如何使用 Gem 管理多角色？

1. **開啟 Gemini 網頁** → 左側導覽列找到「**Gem**」區塊
2. **選擇現有 Gem** → 點選任一 Gem（如「程式說明書助理」），該對話會自動套用該角色的系統提示詞
3. **釘選常用 Gem** → 點擊 Gem 右側的釘書針圖示，可將常用角色釘選在列表上方，方便快速切換
4. **建立自訂 Gem** → 點擊「新的對話」旁的建立按鈕，可建立屬於自己的 Gem，預先寫好角色、風格、規則等系統提示詞

### 為什麼 Gem 適合多角色重複使用？

- **零程式碼**：不需寫程式，在網頁上即可完成
- **一鍵切換**：不同任務時，直接點選對應 Gem，不必重複貼上長串提示詞
- **集中管理**：所有角色集中在左側列表，一目了然
- **可釘選**：常用角色可釘選，減少尋找時間

> 💡 **小技巧**：建立自訂 Gem 時，可參考[系統提示詞範例](../系統提示詞/README.md)的結構，將角色、任務、規則、輸出格式都寫進去，之後每次使用該 Gem 就會自動套用。

## ChatGPT Desktop 的管理方式

**ChatGPT**（網頁版與桌面版）透過 **GPTs（自訂 GPT）** 來管理多角色與重複使用提示詞，概念與 Gemini 的 Gem 類似。

### 什麼是 GPTs？

**GPTs** 是 ChatGPT 的自訂版本，每個 GPT 都有獨立的 **Instructions（系統提示詞）**、知識庫與能力設定。建立多個 GPT 後，可在左側導覽切換使用，等同於切換不同角色。

### 如何建立與管理 GPTs？

1. **進入 GPT 編輯器** → 前往 [chatgpt.com/gpts](https://chatgpt.com/gpts) 或 [chatgpt.com/gpts/editor](https://chatgpt.com/gpts/editor)，點選「**+ 建立**」
2. **在 Create 分頁描述需求** → 與 GPT Builder 對話，例如：「做一個程式碼審查專家，幫我檢查 Python 程式碼的風格與安全性」
3. **在 Configure 分頁細部設定**：
   - **Instructions**：詳細的系統提示詞（角色、任務、規則、輸出格式）
   - **Prompt Starters**：預設的對話開場提示，方便快速開始
   - **Knowledge**：上傳檔案作為知識庫（最多 20 個檔案）
   - **Capabilities**：啟用網頁搜尋、圖像生成、程式碼解析等
4. **儲存與分享** → 選擇「Publish」，可設為僅自己使用或分享給他人

### ChatGPT 多角色管理特色

| 特色 | 說明 |
|------|------|
| **Instructions** | 每個 GPT 有獨立的系統提示詞，切換 GPT 即切換角色 |
| **知識庫** | 可上傳檔案，GPT 會參考這些內容回答，適合專案專用資料 |
| **Prompt Starters** | 預設對話開場，一鍵開始常見任務 |
| **版本管理** | 可查看 Version History、複製或刪除 GPT |

> ⚠️ **注意**：建立與編輯 GPT 需 **ChatGPT Plus、Team 或 Enterprise** 方案；目前僅支援網頁版建立，桌面版可正常使用已建立的 GPT。

## Claude Desktop 的管理方式

**Claude**（網頁版與桌面版）透過 **Projects（專案）** 來管理多角色與重複使用提示詞，每個專案有獨立的系統提示詞與知識庫。

### 什麼是 Projects？

**Projects** 是 Claude 的專案空間，每個專案可設定：
- **專案自訂指令**：該專案專用的系統提示詞
- **釘選文件**：上傳並固定參考文件，Claude 會在對話中參考
- **獨立對話歷史**：每個專案的對話互不干擾

### 如何建立與管理 Projects？

1. **進入專案頁面** → 前往 [claude.ai/projects](https://claude.ai/projects)，點選「**+ 新增專案**」
2. **命名與設定** → 輸入專案名稱、描述，選擇可見性（私密或共享）
3. **設定專案指示** → 點擊「**設定專案指示**」，寫入該專案專用的系統提示詞（角色、風格、規則等），儲存後該專案內所有對話都會自動套用
4. **釘選文件（選用）** → 上傳 PDF、TXT、MD、DOCX、程式碼等檔案，Claude 會參考這些內容回答

### Claude 多角色管理特色

| 特色 | 說明 |
|------|------|
| **專案自訂指令** | 每個專案有獨立的系統提示詞，切換專案即切換角色 |
| **釘選文件** | 單檔最大 30MB，專案總容量 200MB，適合專案專用知識庫 |
| **獨立對話** | 不同專案的對話歷史分開，方便依任務分類 |
| **團隊協作** | Teams 版本可與團隊成員共享專案 |

### 常見專案範例

- **程式碼審查專家**：設定關注類型安全、效能、安全漏洞等
- **技術寫作編輯**：設定風格要求、目標讀者、格式偏好
- **學習伙伴**：設定教學風格、用類比解釋、提供練習題

> 💡 **小技巧**：建立專案時，可依「工作類型」或「角色」命名（如「客服回信」「會議記錄」「程式教學」），方便快速切換情境。

## 三種工具比較

| 比較項目 | Gemini（Gem） | ChatGPT（GPTs） | Claude（Projects） |
|----------|---------------|----------------|-------------------|
| **核心概念** | 可重複使用的角色模板 | 自訂 GPT，含 Instructions | 專案空間，含專案指示 |
| **系統提示詞** | 每個 Gem 獨立 | 每個 GPT 的 Instructions | 每個專案的專案指示 |
| **知識庫** | 依 Gem 而定 | 每 GPT 最多 20 個檔案 | 釘選文件，專案總容量 200MB |
| **切換方式** | 左側點選 Gem | 左側切換 GPT | 左側切換專案 |
| **建立入口** | 新的對話旁建立按鈕 | chatgpt.com/gpts → + 建立 | claude.ai/projects → + 新增專案 |
| **付費需求** | 依 Gemini 方案 | Plus / Team / Enterprise | Pro / Teams |

## 🔥 撰寫系統提示詞的實用技巧

| 技巧 | 說明 |
|------|------|
| **先定義角色** | 用「你是……」開頭，讓 AI 進入正確的「人設」 |
| **條列規則** | 用「要／不要」「必須／禁止」清楚區分，AI 更容易遵守 |
| **指定輸出格式** | 若需要固定格式（JSON、Markdown、表格），在系統提示詞裡寫清楚 |
| **限制範圍** | 明確寫出「只處理……」「不回答……」可避免 AI 偏題 |
| **指定語言** | 若需要特定語言，直接寫「請使用繁體中文」等 |

## 🚀 課堂練習：設計你的系統提示詞

請從以下情境中選一個，試著寫出一段 **50～100 字** 的系統提示詞：

1. **英文作文批改助手**：專門幫高中生批改英文作文，指出文法錯誤並給建議。
2. **程式碼註解助手**：幫開發者為程式碼加上中文註解，風格要簡潔專業。
3. **讀書摘要助手**：幫使用者把長篇文章整理成 3～5 點重點摘要。

> 💡 **提示**：記得包含「角色」「任務」「規則」三個要素！
