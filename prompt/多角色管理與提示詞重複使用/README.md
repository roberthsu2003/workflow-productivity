# 多角色管理與提示詞重複使用

當你需要**多個不同角色**（例如：客服助手、會議記錄助手、文件整理助手），又希望**重複使用**這些提示詞、不用每次重寫時，主流 AI 工具都提供對應功能。以下分別說明 **Gemini**、**ChatGPT**、**Claude** 的管理方式。

---

## 一、Gemini 的 Gem 功能

### 什麼是 Gem？

**Gem** 是 Gemini 網頁版內建的「可重複使用的角色模板」。你可以把它想像成一個**預先設定好個性的助理**，每次點選就能直接使用，不用重新介紹自己需要什麼。

> 📌 **白話說明**：就像公司裡有不同的同事，一位專門處理客訴、一位專門寫報告——Gem 就是你替 AI 建立的這些「專職同事」。

### 常見的 Gem 範例

| Gem 名稱 | 適合用途 |
|----------|------|
| 資訊圖表規畫師 | 規劃簡報或海報的結構 |
| 程式說明書助理 | 協助撰寫操作說明文件 |
| 迭代 Prompt 生成器 | 協助優化你給 AI 的指令 |
| 會議記錄整理助手 | 將會議內容整理成條列重點 |

### 如何使用 Gem？

1. 開啟 [gemini.google.com](https://gemini.google.com)
2. 在左側導覽列找到「**Gem**」區塊
3. 點選想要的 Gem，即可開始對話
4. 點擊右側「📌 釘書針」圖示，可將常用 Gem 固定在最上方
5. 點擊「**建立新的 Gem**」，可以自己寫系統提示詞、打造專屬助理

### ✏️ 小練習：建立你的第一個 Gem

> **情境**：你是行政人員，每週都要整理會議記錄，每次都要重新告訴 AI 你的需求。
>
> 請試著描述：如果你要建立一個「會議記錄整理助手」的 Gem，你會希望它具備哪些特質？（例如：語言、格式、重點項目）

<details>
<summary>💡 參考解答</summary>

以下是一段可用於 Gem 的系統提示詞範例：

```
你是一位專業的會議記錄整理助手。
請使用繁體中文，將我提供的會議內容整理成以下格式：
1. 會議主題
2. 出席人員
3. 討論重點（條列式）
4. 待辦事項（標明負責人與期限）
語氣請簡潔正式，不要加入不必要的說明。
```

**重點提示**：
- 「使用繁體中文」→ 指定語言
- 「整理成以下格式」→ 指定輸出格式
- 「不要加入不必要的說明」→ 限制範圍

</details>

## 二、ChatGPT 的 GPTs 功能

### 什麼是 GPTs？

**GPTs** 是 ChatGPT 的「自訂版本」，每個 GPT 都有自己的角色設定、指令規則，甚至可以附上參考資料。建立多個 GPT 後，隨時從左側切換，就像切換不同的專業顧問。

> 📌 **白話說明**：就像你的手機 App，每個 GPT 都是一個獨立的「AI 應用程式」，設定一次、重複使用。

### 如何建立 GPT？

1. 前往 [chatgpt.com/gpts](https://chatgpt.com/gpts)，點選「**＋ 建立**」
2. 在 **Create 分頁**：用對話方式描述你想要的角色（例如：「幫我做一個回覆客訴信的助手」）
3. 在 **Configure 分頁** 進行細部設定：

| 設定項目 | 說明 |
|----------|------|
| **Instructions** | 完整的角色設定與規則 |
| **Prompt Starters** | 預設開場白，一鍵開始對話 |
| **Knowledge** | 上傳公司文件、FAQ 等參考資料（最多 20 個檔案）|
| **Capabilities** | 啟用網頁搜尋、圖片生成等功能 |

4. 設定完成後點「**儲存**」，選擇「**僅自己使用**」或「**分享給他人**」

> ⚠️ **注意**：建立 GPT 需要 **ChatGPT Plus 以上**方案，免費版無法建立，但可以使用別人公開分享的 GPT。

### ✏️ 小練習：設計 GPT 的 Instructions

> **情境**：公司客服部門每天要回覆大量的詢問信件，內容重複度高，希望建立一個「客服回信助手」GPT。
>
> 請試著寫出這個 GPT 的 Instructions 重點應該包含哪些？

<details>
<summary>💡 參考解答</summary>

以下是 Instructions 的範例架構：

```
你是【OO公司】的專業客服助手。
你的任務是根據客戶來信，撰寫有禮貌、清楚的回覆信件。

【規則】
- 請使用繁體中文
- 語氣要親切但專業，不要過度口語
- 回信開頭請稱呼對方為「您好」
- 若問題超出範圍，請回覆「感謝您的詢問，將由專人為您服務」
- 不要自行承諾任何賠償或期限

【輸出格式】
主旨：Re: [原信主旨]
內文：[回覆內容]
```

**重點說明**：
- 加上公司名稱，讓 AI 知道代表誰說話
- 「不要承諾賠償」→ 這類**禁止事項**非常重要，避免 AI 亂承諾
- 指定輸出格式，回信結構一致

</details>

## 三、Claude 的 Projects 專案功能

### 什麼是 Projects？

**Projects（專案）** 是 Claude 的多角色管理空間。每個專案就像一個獨立的工作資料夾，有自己的角色設定，也可以放入相關文件，讓 AI 在回答時能參考。

> 📌 **白話說明**：就像你電腦裡的資料夾，不同專案的對話和設定完全分開，互不干擾。

### 如何建立 Projects？

1. 前往 [claude.ai/projects](https://claude.ai/projects)，點選「**＋ 新增專案**」
2. 輸入**專案名稱**（例如：「合約審閱助手」「週報整理助手」）
3. 點擊「**設定專案指示**」，寫入這個專案專用的角色設定與規則
4. 可選擇上傳**參考文件**（PDF、Word、程式碼等），Claude 會在對話中參考這些內容

### Claude Projects 的特色

| 特色 | 說明 |
|------|------|
| **專案自訂指令** | 每個專案有獨立設定，切換專案即切換角色 |
| **釘選文件** | 單檔最大 30MB，專案總容量 200MB |
| **對話獨立** | 各專案對話分開，不會互相混淆 |
| **團隊共享** | Teams 版本可與同事共享專案 |

### ✏️ 小練習：規劃你的 Projects 架構

> **情境**：你同時負責三項工作——整理週報、回覆客戶信件、管理出差申請文件。
>
> 請試著規劃：你會建立哪三個 Project？每個 Project 的「名稱」與「主要用途」是什麼？

<details>
<summary>💡 參考解答</summary>

| 專案名稱 | 主要用途 | 可上傳的文件 |
|----------|----------|------|
| 📝 週報整理助手 | 將當週工作紀錄整理成週報格式 | 週報範本.docx |
| 📧 客戶信件助手 | 草擬客戶回信，語氣專業有禮 | 常見問題FAQ.pdf |
| ✈️ 出差申請助手 | 協助填寫出差申請說明、整理費用 | 出差申請規定.pdf |

**小提示**：
- 專案名稱加上 Emoji，在左側列表更容易辨識
- 上傳公司實際的規定文件，AI 的回答會更符合公司規範

</details>

## 四、三種工具快速比較

| 比較項目 | Gemini（Gem） | ChatGPT（GPTs） | Claude（Projects） |
|----------|--------------|----------------|-------------------|
| **核心概念** | 角色模板 | 自訂 AI 應用 | 工作專案空間 |
| **建立難易度** | ⭐ 簡單 | ⭐⭐ 中等 | ⭐ 簡單 |
| **知識庫** | 依 Gem 設定 | 最多 20 個檔案 | 總容量 200MB |
| **切換方式** | 左側點選 | 左側切換 | 左側切換 |
| **付費需求** | 依方案 | Plus 以上才能建立 | Pro 以上 |
| **適合情境** | 快速切換角色 | 建立複雜自訂 AI | 專案導向管理 |

## 五、撰寫系統提示詞的實用技巧

| 技巧 | 說明 | 範例 |
|------|------|------|
| **先定義角色** | 用「你是……」開頭 | 「你是一位專業的行政助理」 |
| **條列規則** | 用「要／不要」清楚區分 | 「不要使用英文」「必須條列重點」 |
| **指定輸出格式** | 明確說明你要的結構 | 「請用表格呈現」「請分三段說明」 |
| **限制範圍** | 避免 AI 偏題 | 「只處理與費用相關的問題」 |
| **指定語言** | 直接寫明 | 「請全程使用繁體中文回答」 |

## 🚀 綜合練習：設計你的專屬助理

請從以下情境選一個，試著寫出一段 **50～100 字** 的系統提示詞，包含「角色」「任務」「規則」三個要素：

1. **📋 會議記錄助手**：將口語化的會議內容整理成正式的會議紀錄，需標明決議事項與負責人。
2. **📧 電子郵件助手**：幫忙草擬各類工作郵件，語氣專業，並自動補上適當的開場與結尾用語。
3. **📊 數據摘要助手**：將複雜的報表數字整理成主管容易閱讀的 3～5 點重點摘要。

<details>
<summary>💡 範例解答（以「電子郵件助手」為例）</summary>

```
你是一位專業的商務電子郵件撰寫助手。
你的任務是根據我提供的情境與重點，幫我草擬完整的工作郵件。

【規則】
- 請使用繁體中文
- 語氣專業有禮，避免過度口語
- 郵件開頭請加上「您好，」或依情境加上稱謂
- 郵件結尾請加上「敬祝商祺」或適當的結尾用語
- 主旨欄請同時提供建議

【輸出格式】
主旨：[建議主旨]
---
[郵件內文]
```

**解析**：
- ✅ 角色：「商務電子郵件撰寫助手」
- ✅ 任務：「根據情境與重點草擬完整郵件」
- ✅ 規則：語氣、語言、開頭結尾格式都寫清楚
- ✅ 輸出格式：明確要求附上主旨

</details>
