# 提示詞不知道怎麼寫？

當你面對空白輸入框，不知道該怎麼開口、或擔心寫得不夠好時，可以試試以下幾種方式。

---

## 方式一：請 AI 提供 prompt 範本

直接告訴 AI 你的目標，請它幫你產出一個可用的 prompt 範本，你再依需求微調即可。

### 小範例

**情境**：你想請 AI 幫你寫一封婉拒廠商的信，但不知道該怎麼下指令。

**你可以這樣問**：

> 我想請 AI 幫我寫一封婉拒廠商報價的信，要禮貌但明確。請給我一個完整的 prompt 範本，我之後可以直接複製使用。

<details>
<summary>💡 參考解答（點擊展開）</summary>

AI 可能會回覆類似這樣的範本：

```
# 角色
你是一位有經驗的採購人員，擅長以得體的方式處理廠商往來信件。

## 任務
請幫我撰寫一封婉拒廠商報價的信件。

### 背景說明（請填入）
- 廠商名稱：
- 拒絕原因：
- 是否希望保留未來合作可能性：是 / 否

## 輸出格式
- 語氣：禮貌、正式、不拖泥帶水
- 結構：感謝來信 → 說明原因 → 表達未來合作意願（若適用）→ 署名
```

你只要把「背景說明」的欄位填好，貼給 AI 就能使用。

</details>

## 方式二：請 AI 用引導詢問方式幫你

讓 AI 主動問你幾個問題，透過一問一答幫你釐清需求，最後產出適合的 prompt。

### 小範例

**情境**：你想用 AI 整理會議記錄，但不太確定要包含哪些內容。

**你可以這樣問**：

> 我想用 AI 整理會議記錄，但我不太確定該怎麼寫 prompt。請用問答的方式引導我，問我幾個關鍵問題（例如：會議類型、需要哪些欄位、輸出格式等），然後根據我的回答幫我產出一個完整的 prompt。

<details>
<summary>💡 參考解答（點擊展開）</summary>

AI 可能會先問你：

1. **會議類型**：是定期例會、專案討論、還是客戶會議？
2. **需要記錄的內容**：只要決議事項，還是也要記錄討論過程、待辦事項、負責人？
3. **輸出格式**：純文字、Markdown、還是表格？
4. **語氣**：正式報告體、還是簡潔 bullet points？

你回答後，AI 會根據你的答案組合成一個專屬的 prompt，例如：

```
你是會議記錄整理助手。請將以下會議筆記整理成結構化記錄。

輸出格式：
- 會議基本資訊（時間、地點、出席者）
- 決議事項（條列）
- 待辦事項（負責人 | 事項 | 截止日，用表格呈現）

語氣：簡潔、條列式。
```

</details>

## 方式三：尋找網頁的 prompt 庫

網路上有許多**現成的 prompt 庫**，可依分類搜尋、一鍵複製，直接套用或稍作修改。

### 常見的 prompt 庫資源

| 資源 | 說明 | 連結 |
|------|------|------|
| **第五 AI 提示詞** | 10 萬+ 提示詞，分類完整（寫作、商業、開發等），一鍵複製 | [prompt.diwuai.com](https://prompt.diwuai.com/prompts) |
| **Prompt Library** | 開源專案，支援中文，可分類瀏覽與搜尋 | [mrxie23.github.io/PromptLibrary](https://mrxie23.github.io/PromptLibrary/) |
| **Studio Global** | 100+ 專家製作提示，支援繁體中文、團隊共享 | [studioglobal.ai](https://www.studioglobal.ai/zh-hant/prompt-library/) |
| **PromptVault** | 500+ 模板，開發者、內容創作者、小企業等分類 | 可搜尋「PromptVault」 |

### 小範例

**情境**：你想找「寫作」相關的 prompt，但不知道從哪個庫開始。

**你可以這樣做**：

1. 打開 [Prompt Library](https://mrxie23.github.io/PromptLibrary/) 或 [第五 AI 提示詞](https://prompt.diwuai.com/prompts)
2. 在搜尋欄輸入「寫作」或「文章」
3. 瀏覽結果，點選感興趣的 prompt
4. 一鍵複製，貼到 AI 對話框試用
5. 依實際需求微調（例如改語氣、加字數限制）

<details>
<summary>💡 參考解答：從庫中找到的「文章改寫」prompt 範例（點擊展開）</summary>

假設你在 prompt 庫中搜尋「文章改寫」，可能會找到類似這樣的範本：

```
請幫我改寫以下文章，要求：
1. 保持原意不變
2. 用更簡潔、易懂的用詞
3. 適合 [目標讀者，例如：一般大眾 / 專業人士] 閱讀
4. 字數控制在原文的 80% 以內

原文：
（在此貼上你的文章）
```

**使用方式**：複製後，把「目標讀者」和「原文」替換成你的內容即可。

</details>

## 方式四：從簡單句開始，再請 AI 幫你擴充

先寫一句最簡單的需求，再請 AI 幫你補齊細節、結構化。

### 小範例

**情境**：你想請 AI 幫你寫一份新人到職注意事項，但只想到「幫我寫新人注意事項」這幾個字。

**你可以這樣問**：

> 我想請 AI 幫我寫一份新人到職注意事項，但我只會寫「幫我寫新人注意事項」。請幫我把這個簡單的句子擴充成一個完整、結構清楚的 prompt，包含角色、任務、需要涵蓋的內容、輸出格式。

<details>
<summary>💡 參考解答（點擊展開）</summary>

AI 可能會產出：

```
# 角色
你是一位熟悉公司流程的資深員工，負責協助新人順利融入。

## 任務
請幫我撰寫一份新人到職的注意事項清單。

### 需要涵蓋的內容
- 第一天報到的流程（報到時間、地點、聯絡人）
- 需要申請的系統帳號（Email、內部系統、差勤系統等）
- 部門日常工作的基本 SOP
- 常見問題與對應窗口

## 輸出格式
- 語氣：親切、易讀，讓新人不會感到緊張
- 結構：用分段標題區分各主題，每個步驟用數字條列
- 若某項需依公司實際情況填寫，請標註「請依貴公司規定填寫」
```

之後你可以依公司實際狀況，在「需要涵蓋的內容」中增減項目。

</details>

## 方式五：參考同類型任務的範例，再改寫

先找到「跟你目標類似」的 prompt 範例（例如本教材的[內容格式](../內容格式/README.md)、[系統提示詞](../系統提示詞/README.md)），照著結構改寫成你的版本。

### 小範例

**情境**：你想請 AI 幫你寫「出差費用申報說明」，手邊有「會議通知信」的 prompt 範例。

**你可以這樣做**：

1. 打開[會議通知信的 Markdown 範例](../內容格式/README.md)
2. 把「會議通知信」的結構（角色、任務、資訊、輸出格式）保留
3. 把內容改成「出差費用申報」相關欄位

<details>
<summary>💡 參考解答（點擊展開）</summary>

改寫後的 prompt 範例：

```markdown
# 角色
你是公司財務部門的行政人員，負責說明費用申報規定。

## 任務
請幫我製作一份出差費用申報的說明文件。

### 需要涵蓋的內容
- 可報銷的費用項目（交通、住宿、餐費等）
- 各項目的金額上限
- 需要檢附的單據種類
- 申報截止日期與送件方式

## 輸出格式
- 語氣：正式、清楚
- 結構：用表格呈現費用上限，其餘用條列式說明
```

**改寫重點**：角色、任務、結構不變，只替換成「出差費用」相關的具體內容。

</details>

## 快速選擇指南

| 情境 | 建議方式 |
|------|----------|
| 完全不知道從何寫起 | 方式二：請 AI 引導詢問 |
| 有目標但不會結構化 | 方式一：請 AI 給範本，或方式四：從簡單句擴充 |
| 想快速套用現成模板 | 方式三：搜尋 prompt 庫 |
| 手邊有類似範例可參考 | 方式五：參考同類型改寫 |

> 💡 **提醒**：無論用哪種方式，產出的 prompt 都建議先試用一兩次，再依實際效果微調，會越來越順手。
