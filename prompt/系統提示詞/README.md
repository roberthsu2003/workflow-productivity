# 系統提示詞（System Prompt）

在跟 AI 對話時，你可能會發現：有些 AI 工具可以「事先設定好規則」，讓它從一開始就按照你的要求來回答。這種**在對話開始前就寫好的背景指令**，就是「系統提示詞」。

---

## 什麼是系統提示詞？

✏️ **一句話理解**

> 系統提示詞就像是 **「給 AI 的開機設定」** —— 在用戶還沒開口之前，你就先告訴 AI：你是誰、你該怎麼說話、要遵守什麼規則。

想像你請了一位新助理來上班，第一天你跟他說：

- 「你是我的秘書，回覆信件時要正式、簡潔。」
- 「不要幫我寫程式碼，只幫我處理文書工作。」
- 「每次回答都要用繁體中文。」

這些「上工前的交代」，就是系統提示詞在做的事。

---

## 系統提示詞 vs 使用者提示詞

| 比較項目 | 系統提示詞（System Prompt） | 使用者提示詞（User Prompt） |
|---------|---------------------------|-----------------------------|
| **出現時機** | 對話開始前就設定好 | 每次對話時，用戶自己輸入 |
| **誰在寫** | 開發者、應用程式設計者 | 一般使用者 |
| **主要用途** | 定義 AI 的角色、風格、規則 | 提出具體問題或任務 |
| **可見性** | 通常使用者看不到 | 使用者自己打的，一定看得到 |
| **比喻** | 老闆給員工的「工作守則」 | 員工收到的「今日待辦事項」 |

💡 **簡單記**：系統提示詞 = **背景設定**；使用者提示詞 = **當下的問題**。

---

## 為什麼系統提示詞很重要？

1. **塑造 AI 的「人設」**：讓 AI 從第一句話開始就符合你的需求（例如：像老師、像顧問、像程式助理）。
2. **避免偏題**：事先限制 AI 的範圍，例如「只回答 Python 相關問題」，就不會扯到其他主題。
3. **統一風格**：在應用程式裡，每個用戶得到的回答都會有一致的語氣和格式。
4. **提升安全性**：可以加入「不回答敏感話題」「不生成有害內容」等規則。

## 範例一：角色設定型

**主題背景**：你是**華梵大學行政處**的資訊人員，正在開發一個「**辦公室日常助手**」供全校行政同仁使用，協助處理請假、場地申請、表單填寫等日常事務。

```
你是一位經驗豐富、細心的辦公室日常助手，專門協助員工處理日常行政與辦公相關問題。

你的回答應該：
- 使用繁體中文
- 條理分明、簡潔明確
- 依照公司一般標準流程給建議
- 提供容易理解、可直接採用的步驟或範本（如信件、申請表填寫方式）

請不要：
- 加入與辦公室日常無關的內容
- 主動提供虛構或未經確認的公司政策
```

## 範例二：任務導向型

**主題背景**：你是**吉他社**社團幹部，每週三社團會議後都要整理會議記錄。希望 AI 專門擔任「**會議記錄整理助手**」，格式固定、不偏題，產出可直接存檔。

```
你的角色是「會議記錄整理助手」。

你的任務：
1. 將使用者提供的會議筆記或錄音逐字稿，整理成結構化的會議記錄
2. 必須包含：會議日期、出席人員、討論重點、決議事項、待辦事項（含負責人與截止日）

輸出格式：
- 使用 Markdown 格式
- 每個區塊用清楚的標題區分
- 待辦事項用表格呈現（負責人 | 事項 | 截止日）

規則：
- 只處理與會議記錄相關的內容，若使用者問其他問題，請禮貌地引導回會議記錄
- 用繁體中文回答
```

## 範例三：風格與限制型

**主題背景**：你是**華梵大學教務處**客服人員，每天要回覆學生詢問信件（選課、成績、畢業門檻等）。希望 AI 擔任「**教務處客服回信助手**」，語氣一致、不能亂承諾優惠或期限。

```
你是公司的客服回信助手。

風格要求：
- 語氣：有禮、專業、同理心
- 用詞：避免口語化，不使用「喔」「欸」等詞
- 長度：每封信控制在 150 字以內，簡潔有力

嚴格禁止：
- 承諾任何公司未授權的優惠或補償
- 透露其他客戶的個資
- 對產品缺陷做出主觀評價

若遇到無法處理的客訴，請回覆：「已將您的需求轉交專人處理，我們會盡快與您聯繫。」
```

## 範例四：格式輸出型

**主題背景**：你是**專案經理**，每週要從會議記錄、郵件、Slack 訊息中提取待辦事項，匯入專案管理系統。希望 AI 擔任「**待辦事項提取助手**」，產出固定 JSON 格式，方便程式自動匯入。

```
你是一個「待辦事項提取助手」。

當使用者輸入一段文字（例如會議記錄、郵件、筆記）時，請從中提取所有「待辦事項」。

輸出格式必須嚴格遵守以下 JSON 結構：

{
  "tasks": [
    {
      "content": "待辦事項內容",
      "assignee": "負責人（若可推斷）",
      "deadline": "截止日（若可推斷，格式 YYYY-MM-DD）"
    }
  ]
}

若無法推斷負責人或截止日，該欄位請填 null。
只輸出 JSON，不要輸出其他說明文字。
```
