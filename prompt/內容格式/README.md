# Prompt 的三大文字格式與實戰技巧

在請 AI 幫忙做事時，你「怎麼說」和「說什麼」一樣重要。同樣一個需求，用不同的格式寫給 AI，得到的結果可能天差地遠。

為了讓大家感受差異，我們用同一個職場情境來示範三種格式：

**主題背景**：你是**華梵大學業務部**行政助理，明天下午兩點有「**季度業績檢討會議**」，地點：三樓會議室，要請業務部和行銷部全體同仁參加，需攜帶季度報告。

> **目標：請 AI 幫我寫一封正式的會議通知信**

---

## 1. 自然語言（Natural Language）

✏️ **概念說明**

就像傳 LINE 給同事一樣，想到什麼就說什麼，不需要任何排版規則。這是最直覺的 Prompt 寫法。

- **優點**：完全零門檻，不需要學任何格式，馬上就能用。
- **缺點**：需求一複雜，指令就容易變成「流水帳」，AI 容易漏掉細節（例如忘記寫會議地點、或語氣不夠正式）。
- **適用場景**：簡單的一次性問答、快速草稿、聊天式諮詢。

💡 **會議通知信範例**：

> 「請幫我寫一封會議通知信，是明天下午兩點的季度業績檢討會議，在三樓會議室，要請業務部和行銷部的同事參加，記得叫他們帶上季度報告。語氣正式一點。」

---

## 2. YAML 格式（YAML Format）

✏️ **概念說明**

YAML 是一種用「縮排」和「項目名稱：內容」來整理資訊的格式，看起來很像填表單。

- **優點**：條理分明，AI 讀起來就像在看結構化的資料表，幾乎不會遺漏任何一個欄位。特別適合需要精確控制細節的任務。
- **缺點**：縮排格式需要稍微練習，寫錯位置會讓 AI 讀不懂結構。
- **適用場景**：需要設定多個條件的任務、定義 AI 的角色與風格、或是需要精確控制輸出格式時。

💡 **會議通知信範例**：

```yaml
角色: 專業的行政秘書
任務: 撰寫一封正式的會議通知信
會議資訊:
  主題: 季度業績檢討會議
  日期: 明天（2025年X月X日）
  時間: 下午 14:00
  地點: 三樓會議室
收件人:
  - 業務部全體同仁
  - 行銷部全體同仁
注意事項:
  - 請與會人員攜帶本季度業績報告
語氣: 正式、簡潔
```

🔥 **進階小技巧：多行內容怎麼寫？** 如果某個欄位的內容很長，可以在冒號後面加上 `|` 符號，AI 就會完整保留你的換行：

```yaml
補充說明: |
  1. 請於會議前一天確認出席狀況。
  2. 如有臨時事務無法出席，請提前告知主管。
  3. 會議預計進行 90 分鐘。
```

## 3. Markdown 格式（Markdown Format）

✏️ **概念說明**

Markdown 是目前最主流的結構化寫法，用幾個簡單符號（`#` 代表標題、`-` 代表清單）就能把需求整理得一目了然。

- **優點**：人看得懂、AI 也最擅長解析。學習成本低，卻能把複雜的需求拆解得非常有條理，是 Prompt 寫作中**最萬用的格式**。
- **缺點**：需要記住幾個基本符號，完全沒用過的人需要幾分鐘熟悉。
- **適用場景**：幾乎所有複雜任務都適用——產出長篇文件、多步驟流程、需要同時兼顧邏輯與細節的工作。

💡 **會議通知信範例**：

```other
# 角色
你是一位經驗豐富的行政秘書，擅長撰寫正式的商務信件。

## 任務
請幫我撰寫一封會議通知信，寄送給業務部與行銷部的全體同仁。

### 會議資訊
- **主題**：季度業績檢討會議
- **時間**：明天下午 14:00
- **地點**：三樓會議室

### 注意事項
- 請與會人員**攜帶本季度業績報告**
- 如無法出席，請在今天下班前告知直屬主管

## 輸出格式
- 信件語氣：正式、有禮、簡潔
- 信件結構：主旨 → 通知內文 → 注意事項 → 署名
- 最後附上一行英文版的會議主旨，方便轉寄給外籍同事
```

## 🚀 課堂練習：換你來試試看！

請從以下 **5 個職場情境** 中挑選 **1 個** 你最有感的，試著用 **Markdown 格式** 寫出一段完整的 Prompt，再交給 AI 執行看看！

🎯 **練習情境 5 選 1**：

1. **請 AI 幫你起草一封婉拒廠商的信**
   - **主題背景**：你是**華梵大學採購處**承辦人，收到「**OO 科技有限公司**」的設備報價，因本次預算有限無法合作，希望保留未來合作可能性。
   - *提示方向*：說明拒絕原因、維持良好關係、語氣要禮貌但不拖泥帶水。

   <details>
   <summary>💡 參考 Prompt</summary>

   <br>

   ```markdown
   # 角色
   你是一位有經驗的採購人員，擅長以得體的方式處理廠商往來信件。

   ## 任務
   請幫我撰寫一封婉拒廠商報價的信件。

   ### 背景說明
   - 廠商名稱：OO 科技有限公司
   - 拒絕原因：本次預算有限，暫時無法合作
   - 希望保留未來合作可能性

   ## 輸出格式
   - 語氣：禮貌、正式、不拖泥帶水
   - 結構：感謝來信 → 說明原因 → 表達未來合作意願 → 署名
   ```

   </details>

2. **請 AI 幫你整理一份會議記錄摘要**
   - **主題背景**：你是**吉他社**社團幹部，剛開完「**3 月社團會議**」，手上有雜亂的會議筆記，需要整理成正式紀錄。
   - *提示方向*：把雜亂的會議筆記整理成「決議事項」和「待辦追蹤」兩個區塊。

   <details>
   <summary>💡 參考 Prompt</summary>

   <br>

   ```markdown
   # 角色
   你是一位細心的行政助理，擅長整理會議資訊。

   ## 任務
   請將以下雜亂的會議筆記，整理成結構清楚的會議記錄摘要。

   ### 原始筆記
   （將你的會議筆記貼在這裡）

   ## 輸出格式
   - 語氣：中性、客觀
   - 結構：
     1. 會議基本資訊（時間、地點、出席人員）
     2. 決議事項（條列式）
     3. 待辦追蹤（負責人 + 截止日期）
   ```

   </details>

3. **請 AI 幫你寫一份新人到職的注意事項清單**
   - **主題背景**：你是**華梵大學資訊處**資深員工，下週有**新人報到**，需要一份「**資訊處新人到職注意事項**」，含報到流程、系統帳號申請、部門 SOP。
   - *提示方向*：包含報到流程、常用系統帳號申請步驟、部門常見的 SOP。

   <details>
   <summary>💡 參考 Prompt</summary>

   <br>

   ```markdown
   # 角色
   你是一位熟悉公司流程的資深員工，負責協助新人順利融入。

   ## 任務
   請幫我撰寫一份新人到職的注意事項清單。

   ### 需要涵蓋的內容
   - 第一天報到的流程
   - 需要申請的系統帳號（Email、內部系統、差勤系統）
   - 部門日常工作的基本 SOP

   ## 輸出格式
   - 語氣：親切、易讀，讓新人不會感到緊張
   - 結構：用分段標題區分各主題，每個步驟用數字條列
   ```

   </details>

4. **請 AI 幫你製作一份出差費用申報說明**
   - **主題背景**：你是**華梵大學財務處**行政人員，需要製作「**出差費用申報說明**」供全校同仁參考，含交通、住宿、餐費上限與單據種類。
   - *提示方向*：列出可報銷的項目、上限金額、需要附上哪些單據、截止日期。

   <details>
   <summary>💡 參考 Prompt</summary>

   <br>

   ```markdown
   # 角色
   你是公司財務部門的行政人員，負責說明費用申報規定。

   ## 任務
   請幫我製作一份出差費用申報的說明文件。

   ### 需要涵蓋的內容
   - 可報銷的費用項目（交通、住宿、餐費）
   - 各項目的金額上限
   - 需要檢附的單據種類
   - 申報截止日期與送件方式

   ## 輸出格式
   - 語氣：正式、清楚
   - 結構：用表格呈現費用上限，其餘用條列式說明
   ```

   </details>

5. **請 AI 幫你寫一則內部公告**
   - **主題背景**：你是**華梵大學資訊處**行政人員，本週六晚上 10:00～週日早上 6:00 進行「**公司內部 ERP 系統維護停機**」，需通知全體員工。
   - *提示方向*：公告標題、維護時間、影響範圍、因應建議、聯絡窗口。

   <details>
   <summary>💡 參考 Prompt</summary>

   <br>

   ```markdown
   # 角色
   你是公司的行政人員，負責對全體員工發布內部公告。

   ## 任務
   請幫我撰寫一則系統維護停機的內部公告。

   ### 公告資訊
   - 影響系統：公司內部 ERP 系統
   - 維護時間：本週六晚上 10:00 ～ 週日早上 6:00
   - 影響範圍：期間無法登入系統、無法提交申請單

   ## 輸出格式
   - 語氣：正式、簡潔，讓員工清楚知道需要注意什麼
   - 結構：公告標題 → 維護時間 → 影響範圍 → 因應建議 → 聯絡窗口
   ```

   </details>

> 💡 **提醒**：
> 寫 Prompt 時，記得想清楚 **「角色（Role）」、「任務（Task）」、「規則與細節（Rules）」** 以及 **「輸出格式（Output Format）」** 這四個要素！
