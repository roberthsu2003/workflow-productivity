# Prompt 的三大文字格式與實戰技巧

在與 AI 溝通時，我們給予指令（Prompt）的格式，會直接決定 AI 輸出的精準度與品質。特別是在進行 Vibe Coding（用語感/直覺請 AI 寫程式） 時，選擇對的格式能讓 AI 更懂你的心！

為了解釋差異，我們統一設定一個目標：「**請 AI 寫一個網頁版的貪食蛇小遊戲**」。來看看三種格式分別會怎麼寫：

---

## 1. 自然語言 (Natural Language)

✏️ **概念說明**：
就像平常跟朋友聊天、交代工程師做事一樣，直接用口語化的文字寫出你的需求。這是最直覺的 Prompt 寫法，著重在「描述感覺」。

- **優點**：零門檻，完全不需要學習任何排版規則，想到什麼就寫什麼。
- **缺點**：當遊戲規則變複雜時，指令容易變成「流水帳」，AI 常常會漏看條件（例如忘記設定死亡機制）。
- **適用場景**：簡單的問答、快速生成基礎原型 (Prototype)。

💡 **貪食蛇範例**：

> 「請幫我寫一個網頁版的貪食蛇遊戲。背景要是黑色的，蛇是螢光綠色，蘋果是紅色的。蛇吃到蘋果會變長，而且分數會加 10 分。如果撞到牆壁或自己的身體就會死掉。請給我完整的 HTML 程式碼，越簡單越好，讓我可以直接點開來玩。」

---

## 2. YAML 格式 (YAML Format)

✏️ **概念說明**：
YAML 是一種資料序列化語言，特色是利用「縮排」和「鍵值對 (Key-Value)」來呈現資料層級。

- **優點**：邏輯極度嚴謹，層次分明。AI 對這種格式的解析能力極高，就像在看設定檔一樣，絕對不會遺漏遊戲規則。
- **缺點**：對沒有程式背景的學生來說有學習門檻，縮排錯誤可能導致結構混亂。
- **適用場景**：定義複雜的 AI 角色、設定嚴格的工作流程、或是當你需要精確控制遊戲數值時。

💡 **貪食蛇範例**：

```yaml
Role: 資深前端遊戲工程師
Task: 開發網頁版貪食蛇遊戲 (Snake Game)
Tech_Stack:
  - HTML5 Canvas
  - Vanilla JavaScript
Game_Rules:
  - 碰到邊界或自身即 Game Over
  - 吃到紅色蘋果，長度 +1，分數 +10
UI_Design:
  Background: "#000000"
  Snake_Color: "#00FF00"
  Apple_Color: "#FF0000"
```

🔥 **進階小技巧：多行文字怎麼寫？**
如果你的輸出要求很長，可以在冒號後面加上 `|` 符號，完美保留換行：

```yaml
Output_Requirement: |
  1. 請提供單一的 HTML 檔案，包含所有 CSS 與 JS。
  2. 程式碼必須加上詳細的中文註解。
  3. 遊戲結束時，必須在畫面上顯示「重新開始」按鈕。
```

---

## 3. Markdown 格式 (Markdown Format)

✏️ **概念說明**：
利用簡單的符號（如 `#` 代表標題、`-` 代表清單）來組織文字。這是目前最主流、泛用性最高的結構化 Prompt 寫法，也是 Vibe Coding 中最常被使用的格式。

- **優點**：兼具了「人類易讀」與「AI 易懂」的雙重優勢。學習成本低，卻能把複雜的遊戲機制與介面拆解得非常有條理。
- **缺點**：需要記住幾個基礎的 Markdown 語法符號。
- **適用場景**：開發完整的應用程式、長篇內容生成、需要兼顧邏輯與細節的複雜任務。

💡 **貪食蛇範例**：

```markdown
# Role
你是一位精通互動遊戲開發的資深前端工程師。

## Task
請使用單一 HTML 檔案（包含 CSS 與 JS），開發一個經典的「貪食蛇遊戲」(Snake Game)。

### Game Mechanics (遊戲機制)
- **基本控制**：使用鍵盤上下左右方向鍵控制。
- **計分規則**：初始長度為 3，每吃到一顆蘋果，長度 +1，分數 +10 分。
- **死亡條件**：蛇頭撞到遊戲邊界（400x400 Canvas），或撞到自己的身體。

### UI / UX Design (視覺與體驗)
- **配色風格**：復古街機風（黑色背景，螢光綠色的蛇，鮮紅色的蘋果）。
- **遊戲狀態**：畫面上方需顯示目前「Score」，死亡時彈出「Game Over」並顯示最終分數。

## Output Format
請提供一段可以直接執行的完整程式碼，不需要額外解釋，並將邏輯模組化加上中文註解。
```

## 🚀 學生隨堂練習：換你來 Vibe Coding！

為了熟悉這三種格式的差異，請從以下 **5 個主題** 中挑選 **1 個** 你最感興趣的，試著用 **「Markdown 格式」** 寫出一段完整的 Prompt，並貼給 AI 讓它幫你把程式碼寫出來！

🎯 **練習主題 5 選 1**：

1. **終極密碼（猜數字遊戲）**
   - *提示方向*：玩家輸入 1~100 的數字，系統要提示「太高了」或「太低了」，直到猜中為止。需要有輸入框和送出按鈕。

2. **經典圈圈叉叉 (Tic-Tac-Toe)**
   - *提示方向*：3x3 的九宮格，玩家與 AI 輪流下（或雙人對戰），需要能判斷誰連成一線獲勝，以及平手的狀況。

3. **實用待辦清單 (To-Do List)**
   - *提示方向*：這是一個網頁小工具。要有可以輸入代辦事項的欄位、新增按鈕。清單上的項目可以「打勾完成」或「點擊刪除」。

4. **專注番茄鐘 (Pomodoro Timer)**
   - *提示方向*：畫面上要有大大的數字倒數計時。預設 25 分鐘專注、5 分鐘休息。需要有「開始」、「暫停」與「重置」按鈕。

5. **打地鼠小遊戲 (Whack-a-Mole)**
   - *提示方向*：畫面上要有幾個洞，地鼠會隨機探出頭，玩家點擊到地鼠就加分。設定遊戲時間為 30 秒，時間到顯示總分。

> 💡 **小提醒**：
> 寫 Prompt 時，記得要想清楚**「角色 (Role)」、「任務 (Task)」、「機制/規則 (Rules)」以及「介面設計 (UI Design)」**喔！
