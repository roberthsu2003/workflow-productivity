## 安裝

### 一、Ollama（地端 LLM）

![](./images/ollama.png)

**官方網站**：[https://ollama.com/download](https://ollama.com/download)

**課程所用模型 `gpt-oss:20b`**：為**純文字**取向，**不支援解析圖片**（無視覺／多模態能力，無法辨識圖片、截圖或掃描檔內容）。本單元僅處理逐字稿與會議文字，與此設定一致；若需圖像相關工作，請另選支援視覺之模型或其他工具。

#### macOS 安裝

1. **方式 A（推薦）**：前往 Ollama 官網下載 `.dmg` 安裝檔，雙擊執行安裝。
2. **方式 B**：於終端機執行以下指令：

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

3. 安裝完成後，開啟 Ollama 應用程式。
4. 於終端機執行以下指令下載課程所需模型：

```bash
ollama pull gpt-oss:20b
```

5. **驗證**：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

#### Windows 安裝

1. 前往 Ollama 官網下載 Windows 安裝檔（`.exe`）。
2. 雙擊安裝檔，依指示完成安裝。
3. 安裝完成後，開啟命令提示字元（CMD）或 PowerShell，執行：

```bash
ollama pull gpt-oss:20b
```

4. **驗證**：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

### 二、Buzz（語音轉文字工具）

![](./images/buzz.png)

**官方下載**：[Buzz 於 SourceForge 的檔案頁](https://sourceforge.net/projects/buzz-captions/files/)

Buzz 為免費開源之離線語音轉文字工具，採用 OpenAI Whisper 模型，支援多種語言。

#### macOS 安裝

1. 前往 SourceForge Buzz 專案頁面。
2. 依電腦架構選擇對應版本（Intel 或 Apple Silicon）。
3. 下載 `.dmg` 檔案，雙擊掛載後將 Buzz 拖曳至「應用程式」資料夾。
4. 首次開啟時，於「設定」中下載 small 模型（課程建議使用）。
5. **驗證**：匯入一段音訊檔，確認可成功產生逐字稿。

#### Windows 安裝

1. 前往 SourceForge Buzz 專案頁面。
2. 下載 Windows 安裝檔（`.exe`）。
3. 執行安裝檔。若出現「未簽署」警告，請點選「更多資訊」→「仍要執行」。
4. 首次開啟時，於設定中下載 small 模型。
5. **驗證**：匯入一段音訊檔，確認可成功產生逐字稿。

## 步驟

**本單元最終要產出的文件**：**會議記錄**（討論過程與重點整理）與 **決議事項**（會中明確作成之決議／待辦等）。以下流程以 [`手動合併_逐字稿.srt`](./資料來源/手動合併_逐字稿.srt)（或你自行合併之單檔）為唯一依據，**僅依逐字稿**，勿臆測未出現之內容。

> **Ollama**：**無法上傳附件**，只能**複製貼上**內容；亦**沒有** ChatGPT Canvas 類畫布。  
> **`gpt-oss:20b`**：**不支援解析圖片**（無法以圖片、截圖作為輸入），請勿期待視覺辨識或讀圖。  
> **地端模型**有**上下文長度**限制；若單次貼上超長，請**分段貼**（例如先貼前半、再貼後半）或換較大上下文之模型／版本。

兩段錄音各轉成一支 SRT 後，於編輯器**手動併成單一檔**（銜接與時間軸自行調整）。本倉庫範例為 [`手動合併_逐字稿.srt`](./資料來源/手動合併_逐字稿.srt)；產出**會議記錄與決議事項**時，請以該檔（或你的合併檔）為定稿依據。

### 1. 使用 Buzz 將錄音轉成文字
  
- 模型可選：`tiny`、`small`、`medium`、`large`（本課程範例曾用 **medium**，轉檔較久，大約為 1 個小時）。
- 產出格式以 **SRT 字幕檔**為主（含時間碼與台詞）；本倉庫示例檔名見上表 **[課程範例：分段 SRT 與手動合併成品](#course-srt-manual-merge)**。

### 2. 手動合併兩段 SRT

1. **順序**：與錄音一致，先 **`_1`**、再 **`_2`**。
2. 以 VS Code 等編輯器開啟兩檔，將第二段文字接到第一段之後；若需**單一合法 SRT**（序號與時間軸連續），在編輯器內調整。給 Ollama 寫**會議記錄與決議事項**時，重點是**內容完整、銜接合理**（步驟 3 會請模型略過時間碼、只依台詞）。
3. 存成你的合併檔，或對照本範例：[`手動合併_逐字稿.srt`](./資料來源/手動合併_逐字稿.srt)。

### 3. 用 Ollama `gpt-oss:20b` 產出會議記錄與決議事項

1. 終端機執行 `ollama run gpt-oss:20b`。
2. 開啟 [`手動合併_逐字稿.srt`](./資料來源/手動合併_逐字稿.srt)（或你的合併檔），**全選複製**；與下方 **提示詞**一併貼入對話（單次貼不下則**分段貼**同一輪需求，或拆成「先貼逐字稿、再貼提示詞」兩則訊息）。若要**另開乾淨對話**重做，步驟相同。
3. 產出請以 **會議記錄** 與 **決議事項** 兩部分為主；**僅依貼上之逐字稿**，勿臆測「上一場次」或其他未載明事項。下方提示詞已請模型略過 SRT 時間碼、只依台詞撰寫。

**會議記錄／決議事項用提示詞（貼入 `gpt-oss:20b`）**

```text
角色：你是協助撰寫正式會議文件之助理，輸出繁體中文。

任務：我會貼上會議**逐字稿**（可能為 SRT，含時間碼；無法附檔）。請**只依本次貼上之台詞內容**（略過時間碼與字幕序號），產出兩部分：
1) **會議記錄**：依討論順序整理重點，包含議題、各方意見要點、共識與歧見（若有）；用語正式、條理清楚，可供存檔備查。
2) **決議事項**：逐條列出逐字稿中**明確作成**之決議、結論、待辦、主責單位或人員、期限（若有提及）。逐字稿**未提及**者請勿捏造，可列為「逐字稿未提及」或略過。

限制：勿臆測「上一場次」「其他會議」或逐字稿未載明之外部情境；勿新增逐字稿中未出現之人名、數字、法條或結論。資訊不足處請標註「未明」或「逐字稿未載明」。

格式：Markdown；分兩大區，標題使用「## 會議記錄」「## 決議事項」。
```
