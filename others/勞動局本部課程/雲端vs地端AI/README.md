## 安裝

### 一、Ollama（地端 LLM）

![](./images/ollama.png)

**官方網站**：[https://ollama.com/download](https://ollama.com/download)

#### macOS 安裝

1. **方式 A（推薦）**：前往 Ollama 官網下載 `.dmg` 安裝檔，雙擊執行安裝。
2. **方式 B**：於終端機執行以下指令：

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

3. 安裝完成後，開啟 Ollama 應用程式。
4. 於終端機執行以下指令下載課程所需模型：

```bash
ollama pull gpt-oss:20b
```

5. **驗證**：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

#### Windows 安裝

1. 前往 Ollama 官網下載 Windows 安裝檔（`.exe`）。
2. 雙擊安裝檔，依指示完成安裝。
3. 安裝完成後，開啟命令提示字元（CMD）或 PowerShell，執行：

```bash
ollama pull gpt-oss:20b
```

4. **驗證**：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

### 二、Buzz（語音轉文字工具）

![](./images/buzz.png)

**官方下載**：[Buzz 於 SourceForge 的檔案頁](https://sourceforge.net/projects/buzz-captions/files/)

Buzz 為免費開源之離線語音轉文字工具，採用 OpenAI Whisper 模型，支援多種語言。

#### macOS 安裝

1. 前往 SourceForge Buzz 專案頁面。
2. 依電腦架構選擇對應版本（Intel 或 Apple Silicon）。
3. 下載 `.dmg` 檔案，雙擊掛載後將 Buzz 拖曳至「應用程式」資料夾。
4. 首次開啟時，於「設定」中下載 small 模型（課程建議使用）。
5. **驗證**：匯入一段音訊檔，確認可成功產生逐字稿。

#### Windows 安裝

1. 前往 SourceForge Buzz 專案頁面。
2. 下載 Windows 安裝檔（`.exe`）。
3. 執行安裝檔。若出現「未簽署」警告，請點選「更多資訊」→「仍要執行」。
4. 首次開啟時，於設定中下載 small 模型。
5. **驗證**：匯入一段音訊檔，確認可成功產生逐字稿。

## 步驟

> **Ollama**：**無法上傳附件**（無法像某些網頁版 AI 那樣附檔），請改為在對話中**複製貼上**逐字稿全文；亦**沒有** ChatGPT Canvas 類畫布。  
> **地端模型**有**上下文長度**限制，兩份逐字稿若合計過長，請改**分段貼上**（先貼 `_1`、再在後續訊息貼 `_2`）或換較大上下文之模型／版本。

### 課程範例：兩段 SRT（資料來源）

本單元已置於 [`資料來源`](./資料來源) 資料夾，可直接下載或於本機路徑開啟：

| 分段 | 檔案（點選連結） |
|------|------------------|
| 第一段 `_1` | [`04討論提案第02案_1_逐字稿.srt`](./資料來源/04討論提案第02案_1_逐字稿.srt) |
| 第二段 `_2` | [`04討論提案第02案_2_逐字稿.srt`](./資料來源/04討論提案第02案_2_逐字稿.srt) |

合併輸入／模型產出可另存於同一資料夾（例如 `04討論提案第02案_合併輸入.srt.txt`）。

### 1. 使用 Buzz 將錄音轉成文字
  
- 模型可選：`tiny`、`small`、`medium`、`large`（本課程範例曾用 **medium**，轉檔較久，大約為 1 個小時）。
- 產出格式以 **SRT 字幕檔**為主（含時間碼與台詞）；本倉庫示例檔名見上表 **[課程範例：兩段 SRT](#課程範例兩段-srt資料來源)**。

### 2. 用 Ollama `gpt-oss:20b` 將兩份逐字稿合併成「一份」完整逐字稿

**建議流程**

1. 確認順序為**先 `_1`、後 `_2`**（與錄音分段一致）。
2. 於文字編輯器開啟兩個 `.srt`，**全選複製內容**（Ollama 只能貼文字，無法上傳檔案）。
3. **終端機**執行 `ollama run gpt-oss:20b`，進入互動後**依序**：
   - 先貼下方 **提示詞**（整段）；
   - 再貼 **第一段 `_1` 的完整內容**；
   - 再貼 **第二段 `_2` 的完整內容**（若一次貼兩段太長，可改成分兩則訊息，見下一段）。
4. 若單次貼上超長被截斷，可改：**第一則**貼提示詞 + 「第一段如下」+ `_1` 全文；**第二則**貼「接續：第二段如下」+ `_2` 全文，並請模型**依對話中兩段**合併輸出。

**本機先合併成單一文字區塊再貼（選用）**

Ollama **不能上傳檔案**，此步驟只是把兩段 SRT **拼成一份文字**，方便你一次全選複製、貼進終端機對話；仍須**貼上**，無法以附檔取代。

請在終端機先切換到本單元目錄 **`雲端vs地端AI`**（即與 `資料來源` 同層），再執行：

```bash
{
  echo "=====【第一段 _1】====="
  cat "資料來源/04討論提案第02案_1_逐字稿.srt"
  echo ""
  echo "=====【第二段 _2】====="
  cat "資料來源/04討論提案第02案_2_逐字稿.srt"
} > "資料來源/04討論提案第02案_合併輸入.srt.txt"
```

之後開啟本機產生之 `資料來源/04討論提案第02案_合併輸入.srt.txt`，**全選複製**，先貼提示詞，再貼（或接續貼）合併後的全文至 `ollama run` 對話。

> 若你另存檔名與上述不同，請自行替換 `cat` 與輸出檔路徑。

> **說明**：SRT **含時間碼**。若你只要「**純文字**一份逐字稿」，請使用下方提示詞（已請模型**擷取台詞、合併為純文字**，不保留時間碼）。若你**必須**輸出單一合法 SRT，須請模型**重新編號字幕序與時間軸**（較長），可再自行加寫在「格式」欄。

**合併用提示詞（RTCCF，貼入 `gpt-oss:20b`）**

```text
角色：你是協助整理會議逐字稿的助理，輸出繁體中文。

任務：我會在對話中貼上同一場次兩段**字幕檔（SRT）全文**（無法附檔），順序為先「_1」後「_2」。請先從 SRT 中**擷取台詞文字**（略過時間碼與序號），再合併成**一份連續的完整純文字逐字稿**：保留所有語意內容，不要寫成摘要；兩段銜接處若語意連貫可僅換行分段。若第二段開頭與第一段結尾明顯重疊，請合併為一段敘述並註明「（重複句已併入）」。

背景：兩段來自同一議案分段錄音，經 Buzz 等工具轉出之 SRT。

限制：不要新增兩段皆未出現的人名、數字、法條或結論；聽打錯字可適度順句讀但勿改變原意；不須輸出開場白，直接從合併後逐字稿開始；**不要**在輸出中重現 SRT 時間碼，除非我另外要求。

格式：純文字或 Markdown；開頭加一行「【完整逐字稿（_1＋_2 合併，純文字）】」。
```

### 3. 取得會議摘要與決議（後續）

在**同一份合併逐字稿**定稿後，可再開新對話，**將該稿全文貼上**（同樣無附檔），請 `gpt-oss:20b` 依該稿整理「重點摘要」「決議事項」（指令須**僅依逐字稿**，勿臆測）。
