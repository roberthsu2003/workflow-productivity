## 三、各單元詳細內容

### 單元一：思維升級——AI 是你的智慧協作夥伴

**教學目標**：學會精準指揮 AI，並理解如何透過 System Prompt 規範 AI 行為。

#### 1. 核心觀念：Copilot ≠ Autopilot

- **Copilot（副駕駛）**：AI 協助決策，人類保有主導權
- **Autopilot（自動駕駛）**：全權交由 AI 執行，風險較高
- 公務應用應採 Copilot 思維，確保 Human-in-the-loop

#### 2. YAML 基礎結構示範

```yaml
角色: 勞動法規專家
任務: 撰寫勞檢通知
語氣: 正式嚴謹
字數限制: 300字內
輸出格式:
  - 標題
  - 事實說明
  - 法規依據
  - 改善期限
```

**實作練習**：將自然語言 Prompt 改寫為 YAML 結構，比較輸出差異。

#### 3. 提示詞黃金 5 要素

引導學員將模糊指令改寫為包含以下 5 點的精準指令：

| 要素 | 說明 | 範例 |
|------|------|------|
| **角色 (Role)** | 設定 AI 身分 | 資深公關、法務人員 |
| **任務 (Task)** | 明確交付事項 | 撰寫新聞稿、公函 |
| **背景 (Context)** | 說明情境與受眾 | 活動亮點、事件原委 |
| **限制 (Constraints)** | 設定邊界條件 | 語氣、字數、禁用詞彙 |
| **格式 (Format)** | 指定輸出形式 | Markdown、表格、列點 |

#### 4. System Prompt 的重要性

- **定義**：User Prompt 是每回合的指令；System Prompt 則是對話前的「大腦設定」或「操作守則」。
- **應用價值**：
  - **一致性**：確保 AI 在長對話中始終保持既定身分（如「勞動法規專家」）
  - **安全性 (Guardrails)**：設定邊界，例如「若詢問非本局業務，請禮貌婉拒」「絕對禁止捏造法條」，對公務應用至關重要

### 單元二：行政文書與視覺優化實戰

#### 1. 公文與文案生成（約 25 分鐘）

**實作項目**：
- 運用 5 要素撰寫勞動檢查通知／新聞稿
- 多維度語氣與情境轉換（3 選 1 練習）

| 情境類型 | 練習內容 |
|----------|----------|
| **A. 正式度切換** | 生硬法規條文 → 社群懶人包（白話）vs. 正式函稿（嚴謹） |
| **B. 態度強弱** | 違規事項 → 柔性勸導單（初犯）vs. 嚴正警告信（累犯） |
| **C. 資訊密度** | 同一份報告 → 給長官的 3 句決策摘要 vs. 給承辦人的詳細執行步驟 |

#### 2. 視覺化輔助（約 25 分鐘）

- 將法規文字轉為簡報大綱（Gamma / ChatGPT）
- 生成宣傳圖卡（Bing Image Creator）

#### 3. YAML 與 Markdown 語法介紹（約 10 分鐘）

**為什麼要使用結構化格式？**
- 提升 AI 輸出穩定性
- 降低格式混亂與遺漏欄位風險
- 方便後續自動化處理（如匯入系統或轉成報表）

**Markdown 基礎語法**：標題 (`#`)、列點 (`-` / `1.`)、表格、程式區塊 (` ``` `)

#### 進階練習：語氣轉換

- **情境 A**：選取《勞基法》條文，改寫為「發給事業單位的正式函文」與「發布於官方社群的溫馨提醒」
- **情境 B**：模擬企業缺失場景，撰寫「限期改善通知書」（堅定但給予機會）與「裁處前最後通牒」（嚴厲且強調法律後果）
- **視覺化**：配合上述內容，生成宣導配圖或簡報草稿

### 單元三：會議記錄與訪談神器

#### 1. Google NotebookLM 實戰

- **實作**：匯入勞動法規 PDF，進行跨文件問答（RAG 概念）
- **實作**：匯入會議錄音／訪談逐字稿，一鍵生成摘要與待辦事項

#### 2. 語音轉文字技巧

- 介紹 Whisper 模型概念（精準度）
- 如何處理多人對話與雜訊

#### 操作重點：RAG 體驗

**檢索增強生成 (RAG) 實作**：
1. 上傳《勞動基準法》PDF
2. 提問範例：「請問依據這份文件，加班費計算基準為何？」
3. 展示 NotebookLM 標註的「來源出處」，強調有憑有據，降低 AI 幻覺疑慮

**會議錄音轉摘要**：
1. 上傳模擬的勞資爭議調解會議錄音
2. 請 AI 自動生成：① 爭議焦點 ② 雙方訴求 ③ 初步結論建議

### 單元四：資安關鍵——雲端 vs. 地端 AI

#### 1. 觀念解析

- 為什麼機密不能丟 ChatGPT？（模型訓練機制）
- 機敏資料應使用地端軟體，避免外流

#### 2. 現場演示 (Demo)

- 使用 Ollama 介面,載入 gpt-oss:20b 模型
- **關鍵動作**：現場拔除網路線
- 輸入模擬的敏感測試資料（如去識別化的薪資單），證明 AI 仍可運作且資料完全未外傳
- 說明：涉及個資（姓名、身分證號）的資料，嚴禁輸入一般雲端 AI 工具

#### 3. 選擇指南

| 資料類型 | 建議工具 |
|----------|----------|
| 一般行政文書 | 雲端 AI（ChatGPT 等） |
| 機敏、個資相關 | 地端 AI（Ollama、LM Studio 等） |

## 附件：工具安裝指南

以下為課前必備工具之安裝方式，請學員於上課前依自身作業系統完成安裝與驗證。

### 一、Ollama（地端 LLM）

**官方網站**：<https://ollama.com/download>

#### macOS 安裝

1. **方式 A（推薦）**：前往 [Ollama 官網](https://ollama.com/download) 下載 `.dmg` 安裝檔，雙擊執行安裝。
2. **方式 B**：於終端機執行以下指令：
   ```bash
   curl -fsSL https://ollama.com/install.sh | sh
   ```
3. 安裝完成後，開啟 Ollama 應用程式。
4. 於終端機執行以下指令下載課程所需模型：
   ```bash
   ollama pull gpt-oss:20b
   ```
5. 驗證：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

#### Windows 安裝

1. 前往 [Ollama 官網](https://ollama.com/download) 下載 Windows 安裝檔（`.exe`）。
2. 雙擊安裝檔，依指示完成安裝。
3. 安裝完成後，開啟命令提示字元（CMD）或 PowerShell，執行：
   ```bash
   ollama pull gpt-oss:20b
   ```
4. 驗證：執行 `ollama run gpt-oss:20b`，若可正常對話即表示安裝成功。

### 二、Buzz（語音轉文字工具）

**官方下載**：<https://sourceforge.net/projects/buzz-captions/files/>

Buzz 為免費開源之離線語音轉文字工具，採用 OpenAI Whisper 模型，支援多種語言。

#### macOS 安裝

1. 前往 [SourceForge Buzz 專案頁面](https://sourceforge.net/projects/buzz-captions/files/)。
2. 依電腦架構選擇對應版本（Intel 或 Apple Silicon）。
3. 下載 `.dmg` 檔案，雙擊掛載後將 Buzz 拖曳至「應用程式」資料夾。
4. 首次開啟時，於「設定」中下載 `small` 模型（課程建議使用）。
5. 驗證：匯入一段音訊檔，確認可成功產生逐字稿。

#### Windows 安裝

1. 前往 [SourceForge Buzz 專案頁面](https://sourceforge.net/projects/buzz-captions/files/)。
2. 下載 Windows 安裝檔（`.exe`）。
3. 執行安裝檔。若出現「未簽署」警告，請點選「更多資訊」→「仍要執行」。
4. 首次開啟時，於設定中下載 `small` 模型。
5. 驗證：匯入一段音訊檔，確認可成功產生逐字稿。

### 三、ChatGPT（雲端 AI 對話）

**官方網站**：<https://chat.openai.com/>

1. 前往 [ChatGPT 官網](https://chat.openai.com/)。
2. 點選「登入」或「註冊」。

> **注意**：ChatGPT 為雲端服務，需連線使用。若機關有網路限制，請事先確認可存取 OpenAI 服務。
