# 會議紀錄與摘要

本單元對象為**臺北市政府勞動局**之**工務／工程業務相關同仁**（需整理會議錄音、決議與重點摘要之承辦）。以下為**教學用虛構或去識別化情境**；實務仍須依機關會議規範、個資／資安與錄音管理規定辦理，並經人工審閱。

> 📌 **前置學習**：[AI 提示詞工程指南](../../prompt/AI提示詞工程指南/README.md)。本單元建議 **RTCCF**（**R**ole + **T**ask + **C**ontext + **C**onstraints + **F**ormat）**自然語言分行**，與 [公文與文案的生成](../公文與文案的生成.md) 一致。

> 🔒 **錄音與個資**：僅於機關核定環境處理；勿將未脫敏檔案上傳至未核准之雲端或個人帳號。

**本單元目標**（將**會議錄音**或已轉出之文字，整理為可用於機關紀錄之文字；定稿前須**人工核對**）：

- **① 逐字稿**（或分段稿）
- **② 決議事項**
- **③ 重點摘要**
- 若另備 **簽到與紀錄表影像（PNG）**，可再產出 **④ 出席／簽到對照**、**⑤ 依機關表格欄位套印之會議紀錄草稿**（見下方「有了 PNG 可以多產出什麼」）；資訊量較「僅有錄音」更完整。

**本資料夾附檔**：

- **`.mp3`**：教學／示範用錄音檔，**請保留**
- **`PNG`**：簽到、紀錄表影像，供對照與套表
- **Word 會議紀錄模板**（`會議紀錄模板.docx`）：供 AI 產出之 **Markdown** 對齊欄位後，**複製貼上** Word
- 實際授課可改為**去識別化**範例檔

## 教學範例：兩段錄音檔（分段／續場）＋會議附件（PNG）＋ Word 模板

本範例採**同一討論案**之**兩支錄音**（先後順序 **`_1` → `_2`**）：步驟 1 產出**完整逐字稿**；步驟 2 以逐字稿搭配 **`簽到薄.png`** 與 **`01案會議記錄.png`**（會議紀錄表影像）**整合**；步驟 3 於 **ChatGPT Canvas** 內修訂、**上傳** **[會議紀錄模板.docx](./會議紀錄模板.docx)**，產出**欄位對齊模板**之 **Markdown**，以利複製貼入 Word。

> **說明**：本次示範音檔為**第 02 案**兩段；`01案會議記錄.png` 僅供**欄位與套表參考**。正式撰寫時請以貴單位實際案號為準；實質內容仍以**逐字稿**為準。**Word 模板**為機關實際歸檔時之版面；AI 產出請**對齊該檔之章節／表格**，以利貼上。

**典型順序**：① **步驟 1**：兩支錄音 → **完整逐字稿**（無需 Word 模板）→ ② **步驟 2**：**逐字稿**＋`簽到薄.png`＋`01案會議記錄.png` → **整合稿**（出席對照、討論、摘要、決議等；**此步不對齊** Word 模板）→ ③ **步驟 3**：**ChatGPT Canvas** 內修訂、**上傳** `會議紀錄模板.docx` → 產出**與模板欄位／章節對齊**之 **Markdown** → ④ **步驟 4**：複製貼上至 Word。若工具**不支援**直接上傳音檔，請先以機關核定之**語音轉文字**分別處理兩檔，再貼入對話並以步驟 1 **校對與分段**。

步驟 3 所附模板請優先使用本資料夾 [`會議紀錄模板.docx`](./會議紀錄模板.docx)；若貴機關另有核定版，請改上傳該檔。

### 有了 PNG 可以多產出什麼？

僅有錄音時，AI 主要還原「說了什麼」；加上兩張圖後，可在**不臆測內容**的前提下，多帶出**結構化與對照用**資訊（實際以圖面可辨識為準，**務必人工校對**）：

| 來源 | 較容易多帶出的資訊 |
|------|---------------------|
| **簽到簿 PNG** | 會議名稱／日期（若表頭有）、**出席者姓名與單位**、簽到欄位是否齊全、應到與實到之對照基礎 |
| **會議紀錄表 PNG** | **機關慣用表格欄位**（案由、討論要點、決議、主辦／期限等）、表上預留空格，利於產出「**依表頭套寫之草稿**」而非僅條列摘要 |

綜合來說：錄音負責**實質內容**；PNG 負責**人名／單位／格式／欄位對齊**，三者併用時，較能一次整理出接近**正式會議紀錄格式**的初稿（仍須簽核前覆核）。

<details>
<summary>步驟 1：兩支錄音檔 → 完整逐字稿（RTCCF）</summary>

本步驟**只要一份完整逐字稿**：依兩段錄音（或兩份轉寫初稿）忠實還原發言，**不**在此步對齊 Word 模板、**不**產出摘要或決議。`會議紀錄模板.docx` 請留待**步驟 3（Canvas）**再上傳。

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：請依我提供之兩段會議錄音（或兩份語音轉文字初稿）產出一份「繁體中文完整逐字稿」。順序為：先「_1」後「_2」。每段開頭標註「【第一段】」「【第二段】」；兩段可接續呈現，須清楚保留分段／段落分界。聽辨不清處標示「（聽辨不清）」，勿臆測補字。請勿輸出重點摘要、決議事項或會議紀錄表格式，僅逐字稿。
背景：臺北市政府勞動局內部工務相關討論（教學用）；同案分兩支錄音。
限制：若我貼上的是轉寫初稿，僅修正錯字、標點、分段與明顯誤聽，不要改寫成摘要或加入初稿未出現之人名、數字、法條。
格式：Markdown；發言者不明時可用「發言者一」「發言者二」占位；可搭配簡短分段標題（例如「## 第一段（_1）」）以利閱讀，**無須**對應任何 Word 模板節次。
```

</details>

<details>
<summary>步驟 2：逐字稿＋簽到簿／會議紀錄 PNG → 整合稿（RTCCF）</summary>

請在對話中一併提供：**步驟 1 之完整逐字稿**（含【第一段】「_1」＋【第二段】「_2」）、影像 **`簽到薄.png`**、**`01案會議記錄.png`**（會議紀錄表影像）。本步產出「**整合結果**」：**無須**上傳亦**無須對齊** `會議紀錄模板.docx`，以便步驟 3 再在 Canvas 對照模板微調。

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：我已貼上完整逐字稿（兩段順序：先「_1」後「_2」），並上傳「簽到薄.png」與會議紀錄表影像「01案會議記錄.png」。請將三者整合為一份 Markdown：**出席／簽到對照**（依簽到簿可辨識欄位）、**案由或議題**、**討論要點**、**重點摘要**（3～5 點）、**決議事項**（若無則註明「無明確決議」）、**主辦／完成期限**（逐字稿未載明則「待確認」）。重複討論請合併敘述。
背景：臺北市政府勞動局內部工務相關討論（教學用）；示範錄音為討論提案第 02 案。
限制：**討論、摘要與決議僅能依逐字稿**，勿臆測未出現之人名、金額、法條或承諾。簽到簿、紀錄表影像僅供**人名、單位、表頭與欄位對照**；辨識不清處標「（圖檔不清）」；與逐字稿不一致處標「（與簽到／發言待核）」。勿因圖檔新增逐字稿未載明之事實。
格式：Markdown；**編排可參考兩張 PNG 所見之表格與欄位**，不必與 Word 模板一致（模板留待步驟 3 處理）。
```

</details>

### 步驟 3：於 ChatGPT Canvas 修訂、上傳模板、產出對齊欄位之 Markdown

在 **ChatGPT** 以 **Canvas** 進行（與一般對話分開之編輯畫面），請約略依下列順序操作：

1. **口語開啟 Canvas**，並說明要處理會議紀錄（例：「請開啟 Canvas，我要整理會議紀錄並對齊機關 Word 模板」）。
2. 將**步驟 2 整合稿**全文**貼入 Canvas**（或請模型依對話前文摘要後置入 Canvas）。
3. 在**同一對話／Canvas 工作區**可上傳檔案時，**上傳** [`會議紀錄模板.docx`](./會議紀錄模板.docx)（或貴機關核定版）。
4. 在 Canvas 內與模型**同畫面**修訂用語、補漏、調整段落；滿意後，用下方提示詞請 AI 輸出**最終版 Markdown**——**標題層級、章節順序、表格欄位須與「會議紀錄模板.docx」一致**，方便您**複製後貼入 Word**。

**介面細節**（如何開 Canvas、是否支援於 Canvas 內上傳 `.docx`）隨產品更新而異，以官方說明為準。若暫無法於 Canvas 上傳模板，可於一般對話先上傳模板並下達相同「格式」要求，再將內容貼回 Canvas 編修。

<details>
<summary>Canvas 內建議提示詞：產出與模板欄位對齊之 Markdown（RTCCF）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：Canvas 內為步驟 2 之「整合稿」。我已上傳「會議紀錄模板.docx」。請在保留實質內容之前提下，將全文改寫為**與該 Word 模板之章節、標題層級、表格欄位名稱與順序完全一致**的 Markdown，供我複製貼到 Word；表格請使用 Markdown 表格呈現。
背景：臺北市政府勞動局內部工務相關討論（教學用）。
限制：不得新增整合稿與逐字稿無據之事實。模板有而文稿無之欄位填「待確認」或「（本欄無）」。請勿輸出與 Word 貼上無關之冗長說明，僅輸出可貼入之主文 Markdown。
格式：Markdown；欄位與模板一一對應；必要時在文首一行註明「以下 Markdown 對齊會議紀錄模板.docx」。
```

</details>

與 Canvas 協作之一般概念，可併參 [公文與文案的生成](../公文與文案的生成.md)（**技巧二：ChatGPT Canvas**）及 [討論方式的內容生成](../../../討論方式的內容生成/README.md)。

### 步驟 4：複製貼上至 Word

1. 開啟本機 **會議紀錄模板** `.docx`（與步驟 3 上傳者一致）。
2. 將**步驟 3** 產出、已對齊欄位之 **Markdown**，**依章節與表格**複製至 Word 對應標題與儲存格。
3. 貼上後檢查字型、框線與分頁（Markdown 與 Word 顯示可能略有差異屬正常）。

---

**檢核**：決議與摘要須可追溯至逐字稿或錄音；貼入 Word 前請確認**與會議紀錄模板**之欄位、案號與簽核欄位無誤；若使用 PNG，**出席欄與表頭**須與圖檔及機關正式紀錄交叉比對；保留原始音檔、逐字稿、模板版本與影像以利查核。
