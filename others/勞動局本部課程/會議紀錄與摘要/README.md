# **會議記錄與訪談神器：Google NotebookLM**

## 範例來源[**衛生福利部中央健保署_全民健康保險藥物給付項目及支付標準共同擬訂會議特材部分第75次(114年3月)會議紀錄及附錄(114.4.8新增) **](https://www.nhi.gov.tw/ch/cp-18105-49ecc-3930-1.html)

## 範例說明:肺動脈高壓處置卡匣健保支付點數調整案

- 來源: 21個會議記錄錄音檔

---

## 目標: 
1. 將21個會議記錄錄音檔轉換為逐字稿
2. 自動建立重點摘要與決議事項
3. 手動建立重點摘要與決議事項

---

### 將21個會議記記錄音檔轉換為逐字稿

**AI工具:** NotebookLM

**使用prompt:**

```
我需要將所有的mp3檔,建立一個逐字稿的markdown檔案
輸出格式範例:


[01主席致詞.mp3]
內容:xxxxx
重要內容: xxxxx
-----
[02參閱並確認上次會議紀錄.mp3]
內容:xxxxx
重要內容: xxxxx

```

### 自動建立重點摘要與決議事項

**AI工具**: NotebookLM **報告按鈕功能**

### 注意事項:

- 報告按鈕功能是使用自動產生的prompt,可以手動產生更適合的prompt

### 手動建立重點摘要與決議事項

**AI工具**: NotebookLM **自訂的prompt功能**

```
撰寫一份綜合性簡報，提煉資料來源的主要主題和觀點。
目的: 建立簡捷的重點摘要與決議事項
輸出格式: markdown檔案
輸出內容:
- 重點摘要
- 決議事項
- 行動事項
- 其他事項

```

###  產生會議重點簡報內容

**AI工具**: NotebookLM **prompt功能**

```
產生一份會議重點簡報內容
```

### 自動產生會議重點簡報

**AI工具**: NotebookLM **簡報功能**


## **一、為什麼公務機關需要 NotebookLM**

在政府單位常見情境：

- 會議錄音整理

- 自動摘要會議
- 萃取決議事項
- 整理行動項目
- 產生會議紀錄草稿
- 比較多場會議
- 生成訪談摘要

👉 適合公部門文件型工作
