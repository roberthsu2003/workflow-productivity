# 會議紀錄與摘要

本單元對象為**臺北市政府勞動局**之**工務／工程業務相關同仁**（需整理會議錄音、決議與重點摘要之承辦）。以下為**教學用虛構或去識別化情境**；實務仍須依機關會議規範、個資／資安與錄音管理規定辦理，並經人工審閱。

> 📌 **前置學習**：[AI 提示詞工程指南](../../prompt/AI提示詞工程指南/README.md)。本單元建議 **RTCCF**（**R**ole + **T**ask + **C**ontext + **C**onstraints + **F**ormat）**自然語言分行**，與 [公文與文案的生成](../公文與文案的生成.md) 一致。

> 🔒 **錄音與個資**：僅於機關核定環境處理；勿將未脫敏檔案上傳至未核准之雲端或個人帳號。

**目標**：將**會議錄音**（或已轉出之文字）整理為 **① 逐字稿（或分段稿）② 決議事項 ③ 重點摘要**。若另備 **簽到與紀錄表影像（PNG）**，可再產出 **④ 出席／簽到對照、⑤ 依機關表格欄位套印之會議紀錄草稿**（見下方「有了 PNG 可以多產出什麼」），資訊量比「僅有錄音」更完整，仍須人工核對後定稿。

本資料夾內 **`.mp3`** 為教學／示範用錄音檔，**請保留**；另附 **PNG** 影像供對照簽到與紀錄表格式。實際授課可改為去識別化範例檔。

## 範例一：單一錄音檔＋會議附件（PNG）

**典型順序**：① **逐字稿**（「步驟 1」）→ ② **決議與摘要**（「步驟 2」）→ ③ **（選用）** 若有簽到與紀錄表 PNG，再請 AI **依表頭整合**（「步驟 3」）。若工具**不支援**直接上傳音檔，請先用機關核定之**語音轉文字**產出初稿，再貼入對話並以步驟 1 做**校對與分段**。

### 有了 PNG 可以多產出什麼？

僅有錄音時，AI 主要還原「說了什麼」；加上兩張圖後，可在**不臆測內容**的前提下，多帶出**結構化與對照用**資訊（實際以圖面可辨識為準，**務必人工校對**）：

| 來源 | 較容易多帶出的資訊 |
|------|---------------------|
| **簽到簿 PNG** | 會議名稱／日期（若表頭有）、**出席者姓名與單位**、簽到欄位是否齊全、應到與實到之對照基礎 |
| **會議紀錄表 PNG** | **機關慣用表格欄位**（案由、討論要點、決議、主辦／期限等）、**第 01 案**在表上的位置與預留空格，利於產出「**依表頭套寫之草稿**」而非僅條列摘要 |

綜合來說：錄音負責**實質內容**；PNG 負責**人名／單位／格式／欄位對齊**，三者併用時，較能一次整理出接近**正式會議紀錄格式**的初稿（仍須簽核前覆核）。

<details>
<summary>步驟 1：單一錄音檔 → 逐字稿（RTCCF）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：請依我提供之會議錄音（或語音轉文字初稿）產出「繁體中文逐字稿」：忠實呈現發言內容；聽辨不清處標示「（聽辨不清）」，勿臆測補字。
背景：臺北市政府勞動局內部工務相關討論（教學用）。我有一支錄音檔；若另上傳簽到簿或會議紀錄表之 PNG，請僅作為對照人名、職稱與欄位之輔助，逐字稿內容仍以錄音／轉寫文字為準。
限制：若我貼上的是轉寫初稿，僅修正錯字、標點、分段與明顯誤聽，不要改寫成摘要或加入初稿未出現之人名、數字、法條。圖檔與錄音不一致時，標註「（與圖檔／紀錄表待核）」勿擅自定案。
格式：Markdown；依語意或時間分段；發言者不明時可用「發言者一」「發言者二」占位。
```

</details>

<details>
<summary>步驟 2：已有逐字稿 → 決議事項與重點摘要（RTCCF）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：請閱讀我提供之逐字稿，整理「決議事項」與「重點摘要」；若無明確決議，請在決議欄標示「無明確決議」。若我另提供簽到簿或會議紀錄表 PNG，決議與摘要仍須可回溯至逐字稿，圖檔僅供核對人名、日期、欄位是否一致。
背景：臺北市政府勞動局內部工務相關討論（教學用）。
限制：僅依逐字稿，勿臆測未出現之人名、金額、法條或承諾。
格式：Markdown；先「重點摘要」（3～5 點），再「決議事項」。
```

</details>

<details>
<summary>步驟 3（選用）：錄音逐字稿＋兩張 PNG → 整合為「會議紀錄表格式」草稿（RTCCF）</summary>

在完成步驟 1、2 後，若已上傳 **簽到簿**與**會議紀錄表** PNG，可用下列提示詞請 AI **依圖表欄位**填寫草稿（內容仍須來自逐字稿；圖檔僅供欄位名稱與出席名單對照）。

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：我已提供「討論提案第 01 案」之逐字稿（及前序產出之決議／摘要）。請另參考我上傳之簽到簿 PNG、會議紀錄表 PNG，輸出一份「依紀錄表欄位排列」之 Markdown 草稿：含出席／簽到對照（依簽到簿可辨識欄位）、案由或議題欄、討論要點、決議事項、主辦／完成期限（逐字稿未載明則填「待確認」）。若某欄在圖檔上無對應空格，請在草稿末註明「表格外補充」。
背景：臺北市政府勞動局內部工務相關討論（教學用）。
限制：討論與決議內容僅能依逐字稿，不可因圖檔臆測新增事實。簽到簿上之人名、單位以圖檔辨識為準，辨識不清處標「（圖檔不清）」。與逐字稿發言者不一致時，加註「（與簽到／發言待核）」。
格式：Markdown；大標依紀錄表常見順序（會議資訊、出席、討論與決議）；表格可用 Markdown 表格呈現。
```

</details>

## 範例二：兩個錄音檔（分段／續場）

**檔名示意**：`04討論提案第02案_1.mp3`、`04討論提案第02案_2.mp3`（先後順序須在指令中說明）。

<details>
<summary>RTCCF 提示詞（複製修改）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：我將提供同案兩段逐字稿（順序：先「_1」後「_2」）。請合併為一份「重點摘要」與「決議事項」；重複討論請合併敘述。
背景：臺北市政府勞動局內部工務相關討論（教學用）。
限制：僅依兩段文稿；未載明處標「（逐字稿未清）」。
格式：Markdown；先註明「兩段逐字稿已合併」，再摘要與決議。
```

</details>

---

**檢核**：決議與摘要須可追溯至逐字稿或錄音；若使用 PNG，**出席欄與表頭**須與圖檔及機關正式紀錄交叉比對；保留原始音檔、逐字稿與影像版本以利查核。
