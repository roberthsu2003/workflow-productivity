# 會議紀錄與摘要

對象：**臺北市政府勞動局** **工務／工程業務**同仁（需整理會議錄音、決議與摘要者）。  
以下為**教學用**虛構或去識別化情境；實務仍須依機關會議規範、個資／資安與錄音管理辦理，並經人工審閱。

> 📌 **前置學習**：[AI 提示詞工程指南](../../prompt/AI提示詞工程指南/README.md)。本單元建議 **RTCCF**（Role／Task／Context／Constraints／Format）**分行撰寫**，與 [公文與文案的生成](../公文與文案的生成.md) 一致。

> 🔒 **錄音與個資**：僅在機關核定環境處理；勿將未脫敏檔案上傳未經核准之雲端或個人帳號。

<a id="goals"></a>

## 學習目標與附檔

### 本單元目標

將**會議錄音**或已轉出之文字，整理成可給機關用的紀錄文字（定稿前須**人工核對**）：

1. **逐字稿**（或分段稿）
2. **決議事項**
3. **重點摘要**

若另有**簽到與紀錄表 PNG**，還可補：

- **出席／簽到對照**
- **依表格欄位整理的會議紀錄草稿**（見 [兩張 PNG 的用途](#png-info)）

### 本資料夾附檔

| 類型 | 說明 |
|------|------|
| **`.mp3`** | 示範錄音，請保留 |
| **`PNG`** | `簽到薄.png`、`02案會議記錄.png`，對照與套表 |
| **`會議紀錄模板.docx`** | **步驟 3**：與下方「單一提示詞」**同一則訊息**附上傳（見 [Canvas 一節](#canvas-finetune)），產出對齊欄位之 Markdown 後貼上 Word |

實際授課可改為**去識別化**範例檔。

<a id="flow"></a>

## 四步驟總覽

示範為**同一案**兩支錄音，順序 **`_1` → `_2`**（討論提案第 02 案，音檔見 [下載附件](#download)）。

| 步驟 | 做什麼 | 重點 |
|------|--------|------|
| **1** | 兩段錄音 → **完整逐字稿** | 只要逐字稿；**不要**模板、摘要、決議 |
| **2** | 逐字稿 + **`簽到薄.png`** + **`02案會議記錄.png`** → **整合稿** | 出席、案由、討論、摘要、決議等；**不對齊** Word 模板 |
| **3** | **ChatGPT Canvas**：**首則**附檔 **`會議紀錄模板.docx`**＋貼**單一提示詞**（含「請開啟 Canvas」）→ 再貼**步驟 2 整合稿** → 討論修改 → 執行提示詞**階段（2）**產出 **對齊模板** 的 Markdown | 方便複製 |
| **4** | 將 Markdown **貼進 Word** | 與步驟 3 使用同一份模板 |

**沒有直接上傳音檔時**：請先用機關核定之**語音轉文字**處理兩檔，再貼上對話，用步驟 1 **校對與分段**。

**模板**：優先用本資料夾 [`會議紀錄模板.docx`](./會議紀錄模板.docx)；有核定版請改上傳核定版。

<a id="png-info"></a>

## 兩張 PNG 的用途

有錄音時，AI 主要還原「說了什麼」。加上 PNG，可在**不亂加內容**的前提下，多補**結構與對照**（以圖面可辨識為準，**務必人工校對**）。

| 來源 | 常能多補的資訊 |
|------|----------------|
| **簽到簿** | 會議名稱／日期（若表頭有）、姓名與單位、簽到是否齊全 |
| **會議紀錄表** | 案由、討論要點、決議、主辦／期限等**欄位與空格**，較好套寫草稿 |

**一句話**：錄音＝實質內容；PNG＝人名、單位、表格式樣；併用較接近正式紀錄初稿（仍須簽核前覆核）。

<a id="step1"></a>

## 步驟 1：完整逐字稿

**本步只產逐字稿**：忠實還原兩段錄音（或兩份轉寫初稿），**不要**對齊 Word 模板、**不要**摘要或決議。`會議紀錄模板.docx` 留到 **步驟 3** 再上傳。

<details>
<summary>點開：RTCCF 提示詞（複製用）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：請依我提供之兩段會議錄音（或兩份語音轉文字初稿）產出一份「繁體中文完整逐字稿」。順序為：先「_1」後「_2」。每段開頭標註「【第一段】」「【第二段】」；兩段可接續呈現，須清楚保留分段／段落分界。聽辨不清處標示「（聽辨不清）」，勿臆測補字。請勿輸出重點摘要、決議事項或會議紀錄表格式，僅逐字稿。
背景：臺北市政府勞動局內部工務相關討論（教學用）；同案分兩支錄音。
限制：若我貼上的是轉寫初稿，僅修正錯字、標點、分段與明顯誤聽，不要改寫成摘要或加入初稿未出現之人名、數字、法條。
格式：Markdown；發言者不明時可用「發言者一」「發言者二」占位；可搭配簡短分段標題（例如「## 第一段（_1）」）以利閱讀。
```

</details>

<a id="step2"></a>

## 步驟 2：整合稿

**請準備**

- 步驟 1 的**完整逐字稿**（含【第一段】`_1`、【第二段】`_2`）
- **`簽到薄.png`**
- **`02案會議記錄.png`**

**本步產出**：一份 **Markdown 整合稿**（出席對照、案由、討論、摘要、決議等）。**不必**上傳、**不必**對齊 `會議紀錄模板.docx`，留給步驟 3。

<details>
<summary>點開：RTCCF 提示詞（複製用）</summary>

```text
角色：你是一位熟諳政府會議紀錄寫法的行政承辦。
任務：我已提供完整逐字稿（兩段順序：先「_1」後「_2」），並上傳「簽到薄.png」與會議紀錄表影像「02案會議記錄.png」。請將三者整合為一份 Markdown：**出席／簽到對照**（依簽到簿可辨識欄位）、**案由或議題**、**討論要點**、**重點摘要**（3～5 點）、**決議事項**（若無則註明「無明確決議」）、**主辦／完成期限**（逐字稿未載明則「待確認」）。重複討論請合併敘述。
背景：臺北市政府勞動局內部工務相關討論（教學用）；示範錄音為討論提案第 02 案。
限制：**討論、摘要與決議僅能依逐字稿**，勿臆測未出現之人名、金額、法條或承諾。簽到簿、紀錄表影像僅供**人名、單位、表頭與欄位對照**；辨識不清處標「（圖檔不清）」；與逐字稿不一致處標「（與簽到／發言待核）」。勿因圖檔新增逐字稿未載明之事實。
格式：Markdown；**編排可參考兩張 PNG 所見之表格與欄位**，不必與 Word 模板一致（模板留待步驟 3 處理）。
```

</details>

<a id="canvas-finetune"></a>

## 在 ChatGPT Canvas 對齊 Word 模板

**步驟 3** 在 **Canvas**（與一般對話分開的編輯畫面）操作。

### 為什麼要先「請它開啟 Canvas」？

在 **ChatGPT** 中，**必須明確請模型開啟 Canvas**，介面才會出現 Canvas；若只貼文字而提示詞內無「請開啟 Canvas」，通常**不會**自動進入 Canvas。與本教學一致的做法是：**首則訊息**即**附檔** `會議紀錄模板.docx`，並貼上下方**單一提示詞**（開頭已含請開啟 Canvas 與兩階段任務）；送出後再貼**步驟 2 整合稿**，與模型討論修改，最後請執行階段（2）產出對齊模板之 Markdown。

### 建議操作順序

| 順序 | 做什麼 |
|------|--------|
| 1 | **同一則訊息**內：**附檔上傳** [`會議紀錄模板.docx`](./會議紀錄模板.docx)（或貴機關核定版），並**貼上下方整段提示詞**（內含「請開啟 Canvas」） |
| 2 | 送出後待 Canvas 開啟，將**步驟 2 整合稿**貼入畫布（若字數多可拆成下一則；或首則已貼整合稿則略） |
| 3 | **與模型來回討論、修改**；當你要求「產出對齊模板之 Markdown」或提示詞中階段（2）時，請模型依已上傳之 **Word 模板**輸出**可複製貼上 Word** 的最終稿 |

**若介面不允許與首則同時附檔**：先上傳 `會議紀錄模板.docx` 並貼提示詞（仍須含請開啟 Canvas），再於下一則貼整合稿。若 **Canvas 無法讀取附件**：先在對話確認模型已「看過」模板結構，再於 Canvas 內沿用相同欄位約定。

<details>
<summary>點開：單一提示詞（請開啟 Canvas＋已上傳 Word 模板＋討論修改＋對齊產出 Markdown）</summary>

**用法**：複製下列全文；在 **ChatGPT** 新增訊息時，**一併附加** `會議紀錄模板.docx`，再貼上並送出。隨後依序貼上**步驟 2 整合稿**，與模型討論修改；需要定稿時請口頭請它執行階段（2），或將階段（2）再貼一次作為收尾指令。

```text
請開啟 Canvas。本則訊息已附上「會議紀錄模板.docx」，請先讀取該檔之章節、標題層級與表格欄位，以便後續對齊。

角色：你是一位熟諳政府會議紀錄寫法的行政承辦。

任務分兩階段，請依序進行：
（1）接下來我會貼上「步驟 2」產出之整合稿（Markdown）。請將內容置於 Canvas，與我討論並依我指示修改用語、段落、補漏或決議／摘要表述；可多次來回直到我滿意。
（2）當我表示可以定稿或請你「對齊模板」時，請在保留我們已確認之實質內容下，將 Canvas 內全文改寫為與**所附 Word 模板**之章節、標題層級、表格欄位名稱與順序**完全一致**的 Markdown，供我複製貼到 Word；表格請使用 Markdown 表格呈現。

背景：臺北市政府勞動局內部工務相關討論（教學用）。

限制：不得新增與整合稿、逐字稿無據之事實。模板有而文稿無之欄位填「待確認」或「（本欄無）」。階段（2）請勿輸出與 Word 貼上無關之冗長說明，僅輸出可貼入之主文 Markdown。

格式：階段（2）請用 Markdown；欄位與模板一一對應；文首可加一行「以下 Markdown 對齊會議紀錄模板.docx」。
```

</details>

<a id="step4"></a>

## 步驟 4：貼上 Word

1. 開啟本機 **`會議紀錄模板.docx`**（與步驟 3 相同那份或貴單位核定版）。
2. 將**步驟 3** 產出的 Markdown，**依章節與表格**貼到 Word 對應標題與儲存格。
3. 檢查字型、框線、分頁（Markdown 與 Word 顯示可能略不同，屬正常）。

<a id="checklist"></a>

## 定稿檢核

- **決議、摘要**能回溯到**逐字稿或錄音**。
- **貼進 Word 前**：欄位、案號、簽核欄與**會議紀錄模板**一致。
- 有使用 **PNG** 時：**出席、表頭**與圖檔及機關正式紀錄**交叉比對**。
- **留存**：原始音檔、逐字稿、使用過的模板版本、影像，以利日後查核。
