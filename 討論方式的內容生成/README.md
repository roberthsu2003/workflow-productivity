# 討論方式的內容生成

當你有一個主題，想產出 **Markdown 格式**的內容，並希望透過與 AI **反覆討論、逐步優化**，可以善用 Gemini、ChatGPT、Claude 的「協作編輯」功能（Canvas／畫布／Artifacts）。**Gemini** 請在 **Chat View** 選取 **Canvas** 工具；**Claude** 請**必須**使用 **Artifact** 模式（詳見下文）。內容不是一次定稿，而是由你與 AI 一起打磨到滿意為止。

> **適合對象**：想用 AI 產出與編輯內容的學習者與職場人士  
> **前置說明**：本單元**首次觸發編輯器**的 prompt 建議採 **RTCCF**（Role, Task, Context, Constraint, Format）之 Markdown 結構；結構化觀念可參考 [AI 提示詞工程指南](../prompt/AI提示詞工程指南/README.md)。**圈選修改、多輪對話**則用**自然語言**即可，不必再套框架。

## 什麼是「討論式編輯器」？

> 一個**可編輯的內容區塊**，出現在對話旁邊。你可以直接改文字、圈選段落請 AI 修改，或透過對話讓內容越改越好。

| 特色 | 說明 |
|------|------|
| **雙欄介面** | 左側對話、右側編輯器，內容與討論並存 |
| **即時編輯** | 直接在編輯器中改字、改段落 |
| **選取討論** | 圈選特定段落，請 AI 針對該段修改或給建議 |
| **迭代優化** | 多輪對話，逐步調整語氣、長度、結構 |

| | **Gemini** | **ChatGPT** | **Claude** |

| **匯出格式** | — | PDF / .md / .docx | — |
| **特別說明** | 以手動選工具為準，勿只依賴「自動跳出」 | 可輸入 `/canvas` 強制開啟 | 本單元以 **Artifact 模式**為強制規範 |

## 產出內容類型對照

協作編輯器除 Markdown 外，也可產出**不同類型的結構化內容**；各工具介面與選項不盡相同，使用前可先確認支援範圍。

### Claude Artifacts 產出類型

| 類型 | 說明 |
|------|------|
| **Apps and websites** | 網頁應用程式、前端介面 |
| **Documents and templates** | 結構化文件、報告、範本 |
| **Games** | 互動式遊戲 |
| **Productivity tools** | 計算機、轉換器、小工具 |
| **Creative projects** | 視覺、藝術類創作 |
| **Quiz or survey** | 測驗、問卷、表單 |
| **Start from scratch** | 從空白自訂 |

![Claude Artifacts 產出類型選單](./images/claude-artifacts-categories.png)

### Gemini、ChatGPT 與 Claude 的差異

| 工具 | 說明 |
|------|------|
| **Gemini Canvas** | 先在 **Chat View** **選取 Canvas 工具**；產出以 Markdown、程式碼為主，通常無獨立「類型選單」 |
| **ChatGPT Canvas** | 以 Markdown、程式碼為主，可含圖表、簡報等，依 prompt 判斷 |
| **Claude Artifact** | **必須**使用 **Artifact 模式**；有預設類型選單時可先選類型再下需求 |

**建議**：需要網頁、遊戲、問卷等**明確類型**時，Claude 可先選類型再下需求；Gemini／ChatGPT 則宜在 prompt 裡**寫清楚產出物**（例如「請產生可互動的網頁」「請做問卷表單」）。更多範例見 [Prompt 範例（RTCCF）](#prompt-範例rtccf)。

## 如何觸發各工具的編輯器？

### 通用原則

| 原則 | 說明 |
|------|------|
| **首次產出用 RTCCF** | Role、Task、Context、Constraint、Format 寫清楚，較容易一次滿足格式與長度需求 |
| **明確說「要產出內容」** | 少只問「什麼是 X」，多說「請寫一篇關於 X 的文章」 |
| **指定格式** | 例如「Markdown」「含標題與條列」 |
| **暗示要編輯** | 例如「方便我編輯」「我要邊看邊改」 |
| **要求足夠篇幅** | 「完整文章」「多個章節」比一句話更容易觸發編輯器 |

### 各工具：編輯區怎麼開？（本單元規範）

<a id="tools-editor-howto"></a>

> **Gemini**：在 **Chat View** 上**選取 Canvas 工具**，再貼上 RTCCF prompt。  
> **Claude**：**必須使用 Artifact 模式**（Artifacts）產出可編輯內容，不可僅用一般對話帶過。  
> **ChatGPT**：可併入「使用畫布」「在 Canvas 中顯示」、或輸入 `/canvas`；長文有時會自動開啟畫布。

| 工具 | 操作／可併入 prompt 的補充句 |
|------|------------------------------|
| **Gemini** | **先選工具**：Chat View → **Canvas**；prompt 內仍可寫「請在 Canvas 中產出」以強調版面 |
| **ChatGPT** | 「使用畫布」「在 Canvas 中顯示」、`/canvas` |
| **Claude** | 介面切到 **Artifact／Artifacts** 模式後再送 prompt；可寫「以 Artifact 顯示可編輯 Markdown」作為補強 |

### 萬用範本（RTCCF）

**情境**：寫一篇結構完整的分享文，主題「**我如何用番茄鐘克服拖延**」，讀者為同班同學，產出後要在編輯器裡與 AI 繼續改。

<details>
<summary>💬 建議 Prompt（RTCCF · 點擊展開）</summary>

```markdown
## Role

你是一位擅長撰寫結構化文章的內容編輯。

## Task

撰寫一篇關於「我如何用番茄鐘克服拖延」的文章，包含：前言、至少三個小節（每節有標題與條列重點）、結語或實務建議。

## Context

- 讀者：同班同學
- 性質：個人經驗式分享，主題須緊扣番茄鐘與克服拖延

## Constraints

- 語言：繁體中文
- 勿虛構明顯與主題無關的經歷；可採「一般性建議＋可替換的範例句」

## Format

- 以 Markdown 標題與條列呈現，結構清晰
- **Gemini**：於 **Chat View** 選取 **Canvas** 工具後再送出。**ChatGPT**：**畫布**顯示（或 `/canvas`）。**Claude**：**必須**使用 **Artifact** 模式，方便直接編輯並討論修改
```

</details>

### 貪食蛇遊戲（HTML／JavaScript）

**情境**：做一個簡單**貪食蛇**網頁遊戲，鍵盤操作，供展示或自學。

<details>
<summary>💬 建議 Prompt（RTCCF）</summary>

```markdown
## Role

你是一位擅長網頁遊戲開發的前端工程師。

## Task

產出一個可於瀏覽器執行的「貪食蛇」網頁遊戲：以 HTML + JavaScript 實作；蛇以方向鍵移動；吃到食物變長並計分；撞牆或撞到自己則結束。

## Context

- 使用情境：課堂展示或本地自學，單機即可執行

## Constraints

- 單一 HTML 檔內含必要 CSS／JavaScript，勿依賴外部套件（除非註明）
- 介面簡潔，適合初學者閱讀程式結構

## Format

- 單一 HTML 檔，可直接用瀏覽器開啟執行
- **Gemini**：於 **Chat View** 選取 **Canvas** 工具後再送出。**ChatGPT**：**畫布**顯示（或 `/canvas`）。**Claude**：**必須**使用 **Artifact** 模式，方便直接編輯
```

</details>

## 在編輯器中如何討論與修改？

### 三工具操作對照

| 操作 | Gemini Canvas | ChatGPT Canvas | Claude Artifacts |
|------|----------------|------------------|------------------|
| **直接改文字** | 點擊後輸入 | 點擊後輸入 | 點擊後輸入 |
| **圈選修改** | 選取 →「選擇並詢問」 | 反白 → 輸入框 | 圈選後於對話說明 |
| **快捷調整** | 語氣、長度等 | 建議編輯、長度、閱讀程度 | 以對話指令為主 |
| **版本還原** | 「上一版本」 | 頂端箭頭 | 左下角版本紀錄 |

### 反覆對話：自然語言即可

編輯器開啟後，**圈選與多輪優化**不必再套 RTCCF，直接說需求即可。例如：「這段改口語一點」「加一個具體例子」「表格加一欄優先級」。

### 四種討論方向（各工具皆適用）

| 類型 | 例句 |
|------|------|
| **要建議** | 「這段結構有沒有更好的寫法？」 |
| **要理由** | 「為什麼要這樣分段？」 |
| **要判斷** | 「這個範例適合初學者嗎？」 |
| **要除錯** | 「這段程式哪裡有問題？」 |

## 完整流程範例：從主題到優化

**情境**：你是**吉他社**幹部，希望有一份統一格式的「**吉他社會議記錄範本**」供每週三會議使用，並能之後在編輯器裡微調。

### 第一步：觸發編輯器（RTCCF）

<details>
<summary>💬 建議 Prompt（RTCCF）</summary>

```markdown
## Role

你是一位擅長會議紀錄整理的行政秘書。

## Task

產出一份「吉他社社團會議記錄範本」，供例行社團會議使用，包含：會議資訊（日期、時間、地點、主席）、出席人員、討論重點、決議事項、待辦事項（含負責人、截止日）。

## Context

- 使用單位：大學吉他社；會議頻率：每週一次為例
- 待辦事項需能以表格呈現

## Constraints

- 語言：繁體中文
- 欄位須符合一般社團會議習慣，勿虛構特定校名規定

## Format

- Markdown，標題與表格清楚，方便後續複製與編輯
- **Gemini**：於 **Chat View** 選取 **Canvas** 工具後再送出。**ChatGPT**：**畫布**顯示（或 `/canvas`）。**Claude**：**必須**使用 **Artifact** 模式，以便直接編輯
```

</details>

### 第二步：圈選後修改（自然語言）

<details>
<summary>💬 範例指令</summary>

```text
這個表格加上「優先級」欄位，並把格式改得更簡潔。
```

</details>

### 第三步：繼續優化（自然語言）

<details>
<summary>💬 範例指令</summary>

```text
決議事項和待辦事項之間，加一個「下次會議時間」區塊。
```

</details>

> 每一輪通常只改你指定的部分，其餘內容維持不變（依各工具行為為準）。

### 練習一：觸發編輯器（基礎）

**情境**：主題「**時間管理的 5 個技巧**」，讀者為大學生，語氣輕鬆實用；目標為成功開啟協作編輯器並產出 Markdown。

**步驟**

1. 開啟 **Gemini**、**ChatGPT** 或 **Claude**（依你選用的工具）  
2. **Gemini**：在 **Chat View** **選取 Canvas 工具**。**Claude**：**必須**切換為 **Artifact** 模式。**ChatGPT**：可直接貼上，或先輸入 `/canvas`／選用畫布（依介面）  
3. 輸入下方建議 Prompt（或保留 RTCCF 結構、替換主題）  
4. 確認出現可編輯內容區塊（Canvas／畫布／Artifact 區）  

<details>
<summary>💬 建議 Prompt（RTCCF）</summary>

```markdown
## Role

你是一位擅長時間管理與生產力提升的教練。

## Task

撰寫一篇「時間管理的 5 個技巧」文章，含前言、五個技巧（各含小標題與簡短說明）、結語。

## Context

- 讀者：大學生
- 語氣：輕鬆、實用

## Constraints

- 語言：繁體中文
- 每個技巧需有獨立小標，並附 2～3 句說明

## Format

- Markdown 標題與條列
- **Gemini**：於 **Chat View** 選取 **Canvas** 工具後再送出。**ChatGPT**：**畫布**顯示（或 `/canvas`）。**Claude**：**必須**使用 **Artifact** 模式
```

</details>

**繳交建議**：截圖標出「對話區」與「編輯器／內容區」。

---

### 練習二：圈選修改（中級）

**情境**：延續練習一產出的文章。需修改兩處：① 其中一個技巧的段落（更口語、加具體例子）；② 結語（加上「今天就開始行動」之呼籲）。其餘不變。

**步驟**

1. 延續練習一文件，或重新用練習一 Prompt 產生一份  
2. 圈選**某一技巧段落**，輸入修改指令（可參考下方）  
3. 再圈選**結語**，輸入第二則指令  

<details>
<summary>💬 建議指令 1（自然語言）</summary>

```text
這段改得更口語、更貼近大學生的生活經驗，並加一個具體例子。
```

</details>

<details>
<summary>💬 建議指令 2（自然語言）</summary>

```text
結語加上一句鼓勵讀者「今天就開始行動」的呼籲。
```

</details>

**繳交建議**：修改前後對照截圖，並簡述圈選範圍與指令。

### 練習三：多輪優化（進階）

**情境**：主題「**113 學年度上學期讀書計畫**」，學期目標為多益 500 分。先產出範本，再經至少 **5 輪對話**逐步加入：SMART 說明、完成度欄位、費曼學習法、活潑語氣、月底回顧區塊。

**步驟**

1. 用下方 Prompt 產生初稿  
2. 至少進行 5 輪修改，每輪用自然語言下指令（可參考下表）  

<details>
<summary>💬 建議 Prompt：初稿（RTCCF）</summary>

```markdown
## Role

你是一位擅長學習規劃與目標設定的教育顧問。

## Task

產出「113 學年度上學期讀書計畫」範本，學期目標為通過多益 500 分；含目標陳述、每週計畫表格、複習策略、自我檢核清單。
```

## 快速參考卡

| 想做的事 | 做法 |
|----------|------|
| **首次觸發編輯器** | 用 **RTCCF** 寫清五段；**Gemini** 先於 **Chat View** 選 **Canvas**；**Claude** **必須**開 **Artifact** 模式（見 [各工具：編輯區怎麼開？](#tools-editor-howto)） |
| **不同產出類型** | 見 [Prompt 範例（RTCCF）](#prompt-範例rtccf) |
| **圈選與多輪修改** | **自然語言**即可，例如：「這段改口語」「加一例」 |
| **沒跳出編輯區** | **Gemini**：確認已選 **Canvas** 工具。**Claude**：確認已用 **Artifact** 模式。**ChatGPT**：試「使用畫布」、`/canvas` 或較長全文 |
| **只改一段** | 先**圈選**該段，再下指令，避免整篇重寫 |

> **重點**：圈選後再下指令，模型較能只改指定片段；這是協作編輯最實用的技巧之一。
