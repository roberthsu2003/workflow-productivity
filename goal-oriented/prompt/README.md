# 二、為什麼 AI 有時候不準？

AI 並不是在「理解指令」，
而是在根據你給的文字去預測最合理的回答。

如果你給的資訊太少，
它就只能用猜的。

範例：

❌ 幫我寫公文

結果通常：空泛、不精準、太籠統。

# 三、三個提升品質的關鍵技巧

## 技巧一：指定角色

你是一位政府機關承辦人。

角色會影響語氣與專業程度。

---

## 技巧二：補充背景

事情是什麼？
對象是誰？
目的為何？

背景越清楚，結果越穩定。

---

## 技巧三：限制輸出格式

**請用正式公文格式撰寫。
字數約300字。
請條列三點**。

格式會讓輸出更好整理與使用。

## 階技巧：使用 Markdown 或 YAML 結構化 Prompt

當任務較複雜時，可以使用 Markdown 或 YAML 讓 Prompt 更有結構。

原因有三：

結構清楚：模型較容易辨識「角色、背景、任務、輸出格式」。

降低誤解：避免指示混在一起造成理解錯誤。

穩定輸出：在多條件、多限制情境下，回答會更一致。

Markdown 範例

```markdown
# 角色
臺北市政府勞動局承辦人

# 任務
撰寫勞動權益宣導公告

# 背景
近期企業未落實加班費規定

# 輸出要求
- 公告格式
- 約200字
- 語氣正式且清楚
```

Markdown 的優點：

以標題分區，閱讀清楚

適合一般使用者快速編輯

適用簡報、文件整理

YAML 範例

```yaml
role: 臺北市政府勞動局承辦人

mission: 撰寫勞動權益宣導公告

background: 近期企業未落實加班費規定

output:
  format: 公告格式
  tone: 正式且清楚
  length: 200字左右
```


YAML 的優點：

條件清單化

適合條件多、限制明確的情境

對未來若導入自動化系統或流程工具特別有幫助

重點觀念：

當任務越複雜，越需要「結構化 Prompt」。

清楚的結構，會讓 AI 的回答更穩定、更專業。
