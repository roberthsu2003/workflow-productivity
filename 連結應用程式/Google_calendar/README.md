# Google Calendar

## 範例：安排課程日期，並產生 xlsx

### AI 工具：Claude.ai

> 使用 **Claude.ai** 連結 **Google Calendar**，以空檔與假日為依據排課，再匯出試算表。

本範例分為 **兩個步驟**，請依序進行：

| 步驟 | 目的 | 產出 |
|------|------|------|
| **步驟一** | 查詢／對照 Calendar，取得**適當排程結論**（空檔、避開日期、開結訓、堂數與時數） | 文字結論＋排程建議（可對照日曆） |
| **步驟二** | 依步驟一結果**建立 Excel 檔案**；優先使用 **Claude.ai 的 Excel／xlsx skill** 直接產出可下載 **.xlsx** | 試算表檔案或欄位與資料列 |

## 步驟一：查詢 Calendar，得到適當排程答案

在 **Claude.ai** 連結 **Google Calendar** 後，請模型**讀取／查詢**你關心的日曆（空檔、既有行程、連假），再搭配下方**任務背景與避開日期**，用 RTCCF 產出**可執行的排程結論**（含建議開訓／結訓、總堂數、每門課 30／33／36 小時之組合）。

**步驟一可做什麼**

- 對照 Calendar 說明**哪些日期不適合排課**、**哪些時段可排**。
- 在符合開結訓與避開日期下，**盡量讓「可以上課的堂數」合計最多**（並與每堂幾小時換算一致）。
- 產出**文字結論＋表格化排程建議**，供步驟二轉成 Excel。

<details>
<summary>步驟一：完整 RTCCF（Markdown，請全選複製）</summary>

```markdown
## Role

你是一位熟悉產投／職訓排課與時數規劃的顧問，能結合日期區間、假日與開結訓限制，並參考我從 Google Calendar 查到的空檔與既有行程，提出可行方案並說明取捨理由。

## Task

在**指定日期區間**內規劃課程；每門課**總時數**僅能為 **30、33 或 36 小時**（每門擇一）。在符合我提供的**時間、開結訓與避開日期**下，**優先使「可以上課的堂數」合計最多**——「堂數」指實際排定之上課**節次／次數**（若我改以「課程門數」或「總授課時數」為優先，以我在 Context 的定義為準）。請在輸出中**明列總堂數**（及各門課拆成幾堂），並標註建議開訓／結訓週次與累計時數，方便對照 Google Calendar。

## Context

- **排程區間（建議與上方文件一致）**：115/9/1～116/2/28；**開訓截止** 115/12/31 前；**結訓截止** 116/2/28 前。
- **每門課時數選項**：僅能為 30、33 或 36 小時（每門課一種）。
- **堂數換算（請補充，至少擇一寫清）**：
  - **每堂（每節）幾小時**（例：每堂 3 小時）；或
  - **每週上幾堂、每堂幾小時**；若未提供，請在輸出先列**假設**再計算（例：固定每堂 3 小時時，單門 30 小時課 ≈ 10 堂）。
- **優先目標**：預設為**整段期間可排定之上課堂數（節次）總和最大**；若與「開課門數最多」或「總授課時數最大」衝突，請在結論說明取捨並依我補充之權重調整。
- **建議避開之連假／節日**（請勿安排課程，與上表相同）：中秋 115/9/25～115/9/28、國慶 115/10/9～115/10/11、光復節 115/10/24～115/10/26、選舉 115/11/28、行憲紀念日 115/12/25～115/12/27、元旦 116/1/1～116/1/3、春節 116/2/4～116/2/10、228 116/2/27～116/2/28。
- **請補充**（無則標「未提供」）：
  - 總可授課時數上限，或每週可排課時數／天數／每天上限
  - 是否同一時段僅能一門課（單一教室／單一師資）
  - 是否允許多門課平行開設
- **Google Calendar**：請依我於對話中提供之**查詢結果**或**空檔說明**（或我請你讀取的日曆範圍）納入排程，勿與既有行程衝突。

## Constraints

- 每門課總時數**只能**是 30、33 或 36 其中之一。
- 開訓、結訓與避開日期**須遵守**上述規定；**不得**臆測未載明的規章或經費。
- 說明**堂數與總時數、每堂時數**的關係：在**每堂時數固定**時，單門課堂數 ≈ 該門總時數 ÷ 每堂時數；**總堂數**為各門堂數加總（或依平行開課規則調整）。若目標是**總堂數最多**，須在「可排時段／可開課門數／每門 30·33·36」之間一併優化，並在結論寫清假設。

## Format

1. **輸入摘要**：條列日期、30／33／36 選項、每堂時數假設、Calendar 相關假設與約束。
2. **結論**：可計算時，給出建議之**總上課堂數（節次）**、各門課之**時數組合與拆成堂數**；必要時附**最大門數**或瓶頸說明。
3. **排程建議**：表格或分段（週次／開訓～結訓／每週幾堂／累計堂數與時數）；**避開**上列日期。
4. **若資訊不足**：標題「待補資訊」，列出問題清單後再給**假設條件下**試算並標明「假設」。
5. **供步驟二使用**：最後附一個**單一 Markdown 表格**，欄位至少包含：`課程序號或名稱`、`課程總時數(30/33/36)`、`拆成堂數`、`建議開訓日`、`建議結訓日`、`備註`，列與列為各門課程，方便複製到 Excel。
```

## 步驟二：建立 Excel 檔案（xlsx）

在 **步驟一** 已定稿（或你認可）的排程結論後，於**同一對話或新對話**請模型整理成試算表。**Claude.ai** 內建 **Excel／試算表相關 skill**（常見為寫入／產出 **.xlsx**），**建議優先使用該 skill 直接產出可下載的 Excel 檔**；若當下無法使用 skill，再改用下方 RTCCF 產出 Markdown 表或 CSV 貼入 Excel。

**步驟二可做什麼**

- **優先**：在 Claude.ai 中**啟用／使用 Excel（xlsx）skill**，請模型依步驟一結論**產生 .xlsx**（可含多工作表、欄寬與表頭格式）。
- **備援**：請模型輸出 **Markdown 表格**、**TSV／CSV**，或貼上步驟一表格後手動匯入 Excel。

<details>
<summary>步驟二：建 Excel 用 RTCCF（Markdown，請全選複製）</summary>

```markdown
## Role

你是一位擅長將排程結論轉成試算表結構的行政助理，產出可供 Excel 使用之欄位與資料列。

## Task

依我提供的**步驟一排程結論**（或上表），建立**一份課程排程試算表**。**若你具備 Excel／xlsx 產出能力（Excel skill）**，請**直接產出可下載的 .xlsx 檔**；否則請以 Markdown 表格或 CSV 提供，供我貼入 Excel。

## Context

- **資料來源**：上一步驟對話中已產出之排程結論與課程列表（若無則請我貼上全文）。
- **輸出格式優先序**：**1)** 使用 **Excel／xlsx skill** 產出 **.xlsx**（含檔名建議，如 `產投課程排程_115下.xlsx`）；**2)** 若無法產檔，再給 Markdown 表或 CSV。
- **欄位需求**（可增列）：`序號`、`課程名稱或代號`、`課程總時數(30/33/36)`、`每堂時數`、`總堂數`、`建議開訓日`、`建議結訓日`、`每週排課摘要`、`備註`、`是否避開連假`。
- **表頭列**：第一列為欄位名稱，**勿合併儲存格**（xlsx 與貼上皆適用）。

## Constraints

- 日期格式一致（建議：YYYY/MM/DD 或西元日期格式）。
- 數字與文字分欄，勿把多個欄位擠在同一儲存格（備註欄除外）。
- 若步驟一未提供某欄資料，該格填「—」或「待補」，勿捏造。

## Format

1. 先給一短段「**試算表說明**」（幾列、用途、是否已用 xlsx skill 產檔）。
2. **優先**：使用 **Excel skill** 交付 **.xlsx**（若平台支援多工作表，可分「摘要」「課程明細」等）。
3. **備援**：輸出**一個完整 Markdown 表格**（可複製貼到 Excel）；或 **CSV 程式碼區塊**（逗號分隔、UTF-8）。
4. 若僅能以 Markdown 表示多工作表：用「**工作表一：…**」「**工作表二：…**」分段，各表一個 Markdown 表格。
```

</details>

> **小提醒**：Claude.ai 的 **Excel／xlsx skill** 可直接產出試算表，請先試「請依上一步結論用 Excel skill 產出 xlsx」；若該輪無法產檔，再將 **Markdown 表格**貼到 Excel，或「資料」→「從文字／CSV」匯入。
