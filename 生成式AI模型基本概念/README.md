# 生成式AI模型基本概念

## 為什麼 ChatGPT 的功能愈來愈多？

### 核心概念：從「模型」到「應用程式」

**ChatGPT 的本質演進：**

| 階段 | 本質 | 功能 |
|------|------|------|
| 最初 | **語言模型** | 輸入文字 → 輸出文字（單純的生成） |
| 現在 | **應用程式** | 文字、圖片、語音、搜尋、檔案處理、外掛程式…… |

---

### 關鍵理解

**1. 底層能力不變**

大型語言模型（LLM）的核心能力始終是：根據輸入產生合理的文字輸出。這就像「引擎」——引擎本身只負責運轉，不決定車子能做什麼。

🏢 **職場案例：** 公司的法務部門同事，他的核心能力是「理解與撰寫法律文件」。不論他是用電話、Email 還是線上系統接案，核心能力都沒變——變的只是溝通與服務的工具。AI 模型也是一樣的道理。

---

**2. 應用程式層的擴充**

ChatGPT 是一個**應用程式**，它把語言模型「包」在裡面。應用程式可以不斷加入新功能，例如：

- 上傳圖片並解讀
- 語音輸入與輸出
- 搜尋即時資訊
- 讀取、分析檔案
- 串接第三方工具（外掛程式）

🏢 **職場案例：** 想像公司的 ERP 系統。一開始只能查庫存，後來加入了採購申請、薪資計算、出差報帳……功能愈來愈多，但背後的「資料庫核心」從來沒換過。ChatGPT 的成長軌跡完全一樣。

---

**3. 比喻說明**

| 比喻元素 | 對應 AI 概念 |
|---------|------------|
| 廚師（會做菜） | **語言模型**（核心能力：理解與生成文字） |
| 餐廳（菜單、點餐、外送、會員） | **應用程式**（ChatGPT、Copilot、Gemini……） |
| 連鎖餐廳加盟擴展 | **API 串接與外掛生態系** |

更貼近職場的比喻：

> **語言模型 = 聰明的新進員工**（有能力、有基礎知識）  
> **應用程式 = 公司的工作流程與系統**（決定這個員工能發揮多大效益）

一個優秀的新員工，進到流程完善的公司，能做的事會遠超出他原本的能力範圍。

---

### 總結

> ChatGPT 功能變多，是因為它是一個**應用程式**。應用程式可以在內部持續整合、擴充各種功能，而底層的語言模型則提供最核心的「理解與生成」能力。

| 功能 | 分類 | 說明 |
|------|------|------|
| 根據提示寫一封 Email | ✅ **模型核心** | 純文字理解與生成，是 LLM 的基本能力 |

| 語音輸入問題、語音回答 | 🔧 **應用程式擴充** | 需要語音辨識（STT）與語音合成（TTS）模組串接 |
| 翻譯一段英文為中文 | ✅ **模型核心** | 語言轉換是 LLM 的核心能力之一 |
| 搜尋今天的股市新聞 | 🔧 **應用程式擴充** | 需要連接外部搜尋引擎或即時資料來源，模型知識有截止日期 |
| 連結公司內部的 CRM 系統 | 🔧 **應用程式擴充** | 需要 API 串接或外掛程式，才能存取外部系統資料 |

> 💡 **教師提示：** 判斷關鍵在於「這件事只需要文字理解與生成嗎？」若需要存取外部資料、感測器或其他系統，就屬於應用程式擴充層。

**範例 1：業務人員 — 客戶報價信件**

| 元素 | 說明 |
|------|------|
| 重複任務 | 每天需依據客戶需求撰寫客製化報價說明信 |
| 廚師（模型核心） | 根據產品資訊與客戶背景，生成一封專業的報價說明文 |
| 餐廳流程（應用程式擴充） | ① 串接 CRM 自動帶入客戶資料 → ② AI 生成草稿 → ③ 業務確認後一鍵發送 |

---

**範例 2：HR 人員 — 面試通知與回覆信**

| 元素 | 說明 |
|------|------|
| 重複任務 | 大量應徵者的面試通知、婉拒信、錄取信 |
| 廚師（模型核心） | 根據職缺名稱與結果，生成語氣適當的信件內容 |
| 餐廳流程（應用程式擴充） | ① 串接應徵系統 → ② 依狀態自動選擇信件類型 → ③ AI 生成 → ④ HR 一鍵審核發送 |

---

**範例 3：行政人員 — 會議紀錄整理**

| 元素 | 說明 |
|------|------|
| 重複任務 | 每次開完會需整理逐字稿成結構化紀錄 |
| 廚師（模型核心） | 將會議逐字稿摘要為「決議事項、待辦事項、負責人」 |
| 餐廳流程（應用程式擴充） | ① 錄音轉文字（語音模組）→ ② AI 整理摘要 → ③ 自動寄送給與會者 |
